네. **기본적으로는 각 Phase의 프롬프트를 복붙하면서 진행하면 됩니다.**
다만 **“교과서 PDF만 첨부하면 끝”은 아닙니다.** Phase가 뒤로 갈수록 **이전 산출물**을 반드시 같이 넣어야 합니다.

이 워크플로우는 단순히 `프롬프트 + PDF` 반복이 아니라,

> **PDF + Scope Lock + 이전 Phase 산출물을 순차적으로 넘겨가며 굴리는 snowball 방식**

입니다. 문서 자체도 Phase별 산출물을 `Step 1 Draft v0 → Audit Report v1 → Crucible Report v1 → Content Lock v2 → Content Lock v2.1 → Draft HTML → Final HTML`로 이어가도록 설계되어 있습니다.

# 가장 효율적인 실행 방식

## 0. 먼저 챕터 단위로 자르세요

가장 중요한 실무 팁은 **교과서 전체 PDF를 매번 넣지 않는 것**입니다.

가능하면:

```text
원본 교과서 전체 PDF
↓
해당 chapter/page range만 추출한 mini PDF
↓
각 Phase에는 이 mini PDF만 첨부
```

이렇게 하세요.

예를 들어 Scope Lock이 `p.120–145`라면, 전체 교과서 800쪽을 넣는 것보다 **p.120–145만 추출한 PDF**를 넣는 것이 훨씬 안정적입니다.
토큰, 시각 인식, figure inventory, source audit 모두 좋아집니다.

# 1. Phase 0 — Scope Lock 먼저 작성

이건 사람이 직접 씁니다.

필수 항목:

```text
Source:
Chapter:
Pages:
Chapter role:
Learner:
Final output:
Mode:
Image rights:
```

여기서 제일 중요한 것은 **Mode 지정**입니다.

| 모드          | 언제 사용                                                                                     |
| ----------- | ----------------------------------------------------------------------------------------- |
| A-Critical  | Clearance, Vd, hepatic extraction, TMDD, FOCE/FO, residual error, IIV, covariate modeling |
| B-Standard  | NONMEM control stream, GOF, simulation, compartment model                                 |
| C-Fast-Safe | 정의·개요·반복 개념·이미 아는 내용                                                                      |

A/B/C 모드에 따라 실행 Phase가 달라집니다. 문서상 A-Critical은 `0→1→2→3→4→4C→5→6`, B-Standard는 `0→1→2→4→4C→5`, C-Fast-Safe는 `0→1→2-lite→4→4C-lite→5`입니다.

# 2. Phase별로 무엇을 첨부해야 하나

| 0     | 없음        | Scope Lock 직접 작성                                                          |

# 5. 매번 PDF를 첨부해야 하나?

## 권장 답

**Phase 1–4C까지는 PDF를 계속 첨부하는 편이 안전합니다.**
단, 전체 교과서 PDF가 아니라 **해당 page range만 자른 PDF**를 쓰세요.

| Phase    |                                    PDF 필요도 |
| -------- | -----------------------------------------: |
| Phase 1  |                                         필수 |

# 8. 실제 운영에서 가장 중요한 체크포인트

## Phase 1 후

다음이 없으면 재실행하세요.

* Curation Map 있음
* §2 Concept Anatomy Cards 있음
* Big Idea가 구체적임
* figure marker가 없음
* HTML 생성 안 함

## Phase 2 후

다음이 있어야 합니다.

* T1–T6 표
* T5 Coverage Audit
* T6 Figure Inventory
* T6에서 figure를 직접 설계하지 않음

문서의 Anti-pattern도 Phase 2에서 “새로운 설명이나 비유가 Audit Report에 포함되면 재지시”, “T6에서 Audit이 figure를 직접 설계하면 inventory only로 재지시”하도록 되어 있습니다. 

## Phase 3 후

다음이 없으면 재실행하세요.

* T4 Deletion Candidates
* T3 insertion text가 1–2문장
* figure 제안 없음

## Phase 4 후

다음이 있어야 합니다.

* Content Lock v1
* Content Lock v1.5
* Content Lock v2
* v2가 v1 대비 8% 초과하지 않음
* 새 사실/새 수치/새 임상 시나리오 없음

## Phase 4C 후

대부분은 **PATCH MODE**를 권합니다.

문서상 Content Lock v2가 길거나 6000 words 초과, 또는 marker 수가 4개 이하이면 PATCH MODE를 권장합니다. PATCH MODE는 본문 재출력에 따른 미세 변형을 막기 위한 장치입니다. 

따라서 실무 기본값은:

```text
Phase 4C는 PATCH MODE 우선
```

입니다.

## Phase 5 후

* HTML 전체가 나왔는지
* marker가 plain text로 남지 않았는지
* Mermaid가 깨지지 않았는지
* Self-Test 정답이 기본 노출되지 않는지
* print 버튼/accordion/nav가 작동하는지

PATCH MODE를 쓴 경우에는 Splice Verification Table이 먼저 있어야 합니다. PATCH MODE에서는 anchor가 없거나 중복되면 HTML 생성을 중단하도록 설계되어 있습니다.

# 결론

**거의 맞습니다.**
하지만 정확히는:

> **각 Phase prompt를 복붙하고, 해당 PDF만 넣는 것이 아니라,
> 해당 PDF + Scope Lock + 이전 Phase 산출물을 누적해서 넣어야 합니다.**

가장 효율적인 방식은:

> **챕터 page range만 자른 mini PDF를 사용하고,
> Phase 4는 같은 GPT-5.5 세션에서 이어서 처리하며,
> Phase 4C는 기본적으로 PATCH MODE를 쓰고,
> Phase 5/6에서 HTML과 QA를 닫는 방식**입니다.

이 방식이면 토큰 낭비를 줄이면서도 v3.3.2의 핵심 장점인 **정확성, 통찰, figure 선별, HTML 완성도**를 모두 살릴 수 있습니다.
