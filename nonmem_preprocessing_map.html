Raw 데이터 수령 & 평가
어떤 형태로 받았는가? → 이후 전체 작업량 결정
핵심 원칙
어떤 형태의 raw 데이터든 최종 목표는 동일: 6개 표준 입력 → 3대 building block (dosing, observation, covariates) 조립 → NONMEM-ready 단일 데이터셋. 소스가 달라도 도착지는 같다.
빈도 최상
SDTM 데이터셋
구성 도메인
DM (demographics), EX (exposure/dosing), PC (pharmacoconcentrations), PP (PK parameters), LB (lab results), VS (vitals), AE (adverse events), CM (concomitant meds), DS (disposition) 등. 파일 형식은 SAS transport (.xpt) 또는 .csv.
장점
CDISC 표준화 → 변수명/구조 예측 가능. 대부분의 제약사/CRO에서 제공하는 표준 형태. 도메인 간 USUBJID로 일관된 merge 가능.
수령 즉시 확인 체크리스트
SDTM version 확인 (3.2 vs 3.3 vs 3.4 → 변수명 차이 존재)
PC 도메인: PCSTAT (completion status), PCORRES (original result), PCSTRESN (numeric standardized result) 존재 여부
EX 도메인: EXSTDTC, EXENDTC datetime 형식 + 실제 투여 기록의 completeness
날짜 형식이 ISO 8601 (YYYY-MM-DDThh:mm)이 아닌 경우 변환 필요
PC에 BLQ 관련 flag가 PCSTAT 또는 PCORRES의 텍스트("BLQ","
핵심 의사결정: EX (dosing) + PC (observations) + DM (baseline covariates) 을 중심으로 조립. LB/VS는 공변량으로 merge. CM은 병용약물 effect 분석 시 활용.
빈도 중
ADaM 데이터셋 (ADSL, ADPC, ADEX 등)
특징
이미 analysis-ready 수준으로 가공된 상태. ADPC에 AVAL (농도), ATPT (timepoint), PARAMCD 등 포함. ADSL에 baseline demographics 총정리.
주의사항
ADaM은 통계 분석(efficacy/safety)용이므로 NONMEM 구조와 다름
Dosing record가 별도 row로 존재하지 않는 경우가 많음 → ADEX에서 dosing 재구성 필요
ADPC의 시간 변수(ATPT, AVISIT)는 nominal time일 가능성 → actual time은 SDTM PC에서 확인
ADaM이 있어도 SDTM을 함께 요청하는 것이 안전. ADaM은 cross-check 및 covariate 추출 용도로 활용. ADSL의 baseline covariates는 직접 활용 가치 높음.
빈도 중
Raw CRF/EDC export
형태
Medidata Rave, Oracle InForm, REDCap 등에서 직접 export한 .csv/.xlsx. 변수명이 CRF page 기반으로 비표준적. 각 form이 별도 파일.
핵심 도전
변수명 매핑 필요 (CRF 변수 → 표준 변수). DDT 없으면 protocol + CRF annotation 참조
날짜/시간이 분리되어 있거나, AM/PM 형식이거나, 다양한 locale 혼재
반복 투여 정보가 schedule table에 별도 존재하거나 누락
Subject ID 체계가 site별로 다를 수 있음
DDT (Data Definition Table)를 반드시 먼저 작성/확보. 없으면 protocol과 CRF를 보고 직접 작성. DDT는 "어떤 소스 변수 → 어떤 NONMEM 변수, 어떤 변환 규칙"을 명시하는 명세서.
빈도 낮음
Investigator Excel / 학술 연구 데이터
특징
연구자가 직접 정리한 스프레드시트. 가장 비표준적. merged cell, color coding, 주석, 여러 시트에 분산, 단위 혼재 빈발.
필수 작업
Merged cell 해제 후 tidy format (1행 = 1관찰) 으로 변환
단위 통일: mg vs μg, L vs mL, hr vs min 등
Subject ID 형식 통일: "001" vs "1" vs "S-001" → 숫자 ID로 재코딩
문헌 데이터의 경우 aggregate (mean±SD) vs individual 구분 확인
가장 오류 가능성 높음. 원본 절대 수정 금지 + 모든 변환을 R 스크립트로 기록 (traceability). readxl::read_excel()로 읽되 col_types 명시적 지정 권장.

상황

핵심 구성요소 조립
3대 building block — 전체 작업의 60-70%
가장 중요
Dosing records (EVID=1 또는 4)
필수 변수
ID TIME AMT CMT EVID=1 MDV=1 DV=0 또는 . — infusion 시 RATE 또는 DUR 추가.
투여 경로별 의사결정 트리
경구 Bolus (가장 흔함): RATE=0 (또는 생략), AMT=투여량(mg). 1st-order absorption은 모델($PK)에서 KA로 처리.
IV Bolus: RATE=0, CMT=central compartment (보통 2). 순간 투입.
IV Infusion (zero-order): 두 가지 옵션 — ① RATE=AMT/주입시간(hr) (양수), ② RATE=-2 + DUR 컬럼에 주입시간 기록. 방법 ②가 데이터에서 직관적.
SC/IM 주사: 보통 bolus로 처리 (1st-order absorption model). 간혹 zero-order + 1st-order 혼합 모델.
반복 투여 패턴
같은 용량, 같은 간격으로 반복: ADDL=추가 투여 횟수, II=투여 간격(hr). 행 수를 대폭 절감. 예: 매일 100mg × 28일 → 1행으로 AMT=100, ADDL=27, II=24
용량 변경 있음: 변경 시점마다 별도 dosing record. ADDL/II는 같은 용량 구간 내에서만 사용.
투여 중단/재시작: 중단 기간 동안 dosing record 없음. 재시작 시 새 dosing record.
Reset 필요? (EVID=4): 체내 약물량을 0으로 초기화하면서 동시에 투여. 예: 2nd course 시작, washout 후 재투여. 주의 — reset은 모든 compartment를 0으로 만듦.
Actual vs Planned dose
원칙: AMT는 반드시 실제 투여된 용량 (actual dose). Protocol에 정해진 planned dose가 아님. Planned dose는 별도 컬럼(PDOSE 등)으로 보존하되, NONMEM에는 actual dose를 사용. 부분 주입, 구토 등으로 실제 투여량이 다를 수 있음.
특수 사례: CAR-T / 세포치료제
세포치료제는 single infusion이지만 체내에서 expansion → contraction 패턴. Dosing은 infusion 시점 1회 (AMT=주입 세포수). 이후 cellular kinetics는 관찰값 (EVID=0)으로 처리. CAR-T 세포수가 DV. 전통적 PK와 다르게 DV가 dose 이후 증가하는 것이 정상.
PK/PD 관찰값
Observation records (EVID=0)
필수 변수
ID TIME DV CMT EVID=0 MDV (0=유효, 1=결측/제외) AMT=0 또는 .
DV (Dependent Variable) 처리
단위 결정 (가장 먼저): 농도 단위를 모델과 일치시킴. AMT(mg) & DV(ng/mL) → CL 단위 = L/hr. 단위 조합이 파라미터 해석에 직결되므로 초기에 확정하고 data specification에 기록.
Log 변환: DV에 직접 ln()을 취해서 데이터에 넣을지, control stream에서 Y=LOG(F)로 처리할지 결정. 일반적으로 control stream에서 처리 권장 (원본 보존). 데이터에서 변환 시 DV=0인 record 처리 주의 (log(0) = -∞).
다중 분석물: parent + metabolite를 동시에 모델링 → 각각 다른 CMT 또는 DVID 할당. DV 단위가 다를 수 있으므로 scaling 필요 여부 확인.
Multiple endpoint (PK + PD) 처리
CMT 기반 구분: CMT=1 → PK 농도, CMT=2 → PD 바이오마커. 각 endpoint의 magnitude가 크게 다르면 residual error model을 endpoint별로 분리해야 하므로 데이터 단계에서 명확히 구분.
DVID 사용 패턴: 일부 모델러는 CMT 대신 DVID 컬럼으로 endpoint 구분 → control stream의 $ERROR에서 DVID에 따라 다른 error model 적용.
Phase 4에서 상세
Covariates (공변량) — 개요
시간불변 (Baseline)
AGE, SEX, RACE, WT(baseline), HT, BSA, 유전형, STUDYID 등. 각 ID당 1개 값.
시간가변 (Time-varying)
CLCR, ALB, ALT, 체중변화, 병용약물 on/off, 질병 상태 변화 등. 시점별로 값 변동.
공변량 전략은 Phase 4에서 상세히 다룸. 여기서는 DM 도메인에서 baseline covariates를 추출하고, LB/VS에서 time-varying covariates를 추출한다는 점만 인지.

TIME 변수 — 가장 치명적인 단계
버그의 #1 원인. 한 번 틀리면 전체 분석이 무의미
최다 오류
TIME 계산 전략
기본 공식
TIME = (해당 이벤트의 datetime) − (reference datetime), 단위는 보통 hours. 모든 dosing + observation + covariate event에 동일 로직 적용.
Reference time 선택
표준 (가장 흔함): 각 ID의 첫 투여 시점 = TIME 0. Predose sample은 음수 또는 0.
대안 — Study Day 1: 이 경우 predose가 음수 TIME. Run-in period의 PD 관찰값도 포함 가능.
절대 규칙: 같은 데이터셋 내 모든 ID가 동일한 reference 로직 사용. multi-study pooling 시에도 study별이 아닌 ID별 첫 투여 기준.
날짜(Date) vs 날짜시간(Datetime)
PK 분석: 반드시 datetime 사용. 채혈-투여 간격이 분 단위로 PK 파라미터에 영향. 시간 정밀도 = 분석 정밀도.
PD 분석 (daily endpoints): AE 발생일, 종양 크기 측정일 등은 날짜만으로 충분한 경우 있음.
혼합 상황: PK+PD 통합 모델이면 datetime 기준 통일. PD 관찰의 시간이 없으면 convention 적용 (예: clinic visit = 오전 08:00 가정).
시간 결측(Missing time) Imputation 규칙
투여 시간 누락: protocol의 투여 스케줄 참조 (예: 아침 08:00 가정). 또는 해당 visit의 첫 기록 시간 활용.
채혈 시간 누락: nominal time (protocol 상 예정 시간) 사용. actual이 없을 때의 최후 수단.
필수: imputed된 시간은 반드시 FLAG 컬럼 (TIMEF 등)으로 표시. Sensitivity analysis에서 imputed records 제외하여 영향 평가.
실무 핵심
시간 관련 함정 & 디버깅
Negative TIME: predose sample이 첫 투여 후로 잘못 배정되면 의도치 않은 음수. 허용될 수 있지만 반드시 의도적이어야 함. QC에서 음수 TIME 목록 출력 후 수동 확인.
같은 TIME에 dose + obs: 허용되지만 정렬 순서가 결정적 — dose record가 반드시 먼저 와야 NONMEM이 "투여 후 관찰"로 해석. 역순이면 "투여 전 관찰"이 됨.
TIME=0인 관찰값: 첫 투여와 동시 채혈인지, predose인지 구분. predose이면 TIME을 -0.01 등 미세 음수로 설정하는 convention 존재.
Timezone 불일치: 다기관 국제 연구에서 각 센터의 timezone 불일치. 보통 local time 기준 통일 (시차 보정 불필요). 단, PK 시료가 중앙 lab에서 기록된 시간이면 주의.
DST (Daylight Saving Time): 장기 연구에서 spring forward/fall back으로 1시간 오차 발생 가능. 드물지만 존재.
Actual vs Nominal time: Actual time 사용이 원칙. Nominal time은 exploratory plot 또는 actual 결측 시 imputation용. 데이터에 둘 다 보존 권장 (TIME = actual, NTIME = nominal).
R datetime 파싱 주의
lubridate::ymd_hms() 등 사용 시 — timezone 인자 명시 (tz="Asia/Seoul" 등). 미지정 시 시스템 locale 따라감. as.numeric(difftime(..., units="hours"))로 TIME 계산. Date class와 POSIXct 혼합 연산 시 예기치 않은 결과 발생.

BLQ & 특수 데이터 처리
정량한계 미만, 이상치, 정상상태 등
분석 영향 大
BLQ (정량한계 미만) 처리 — M1~M7
방법 비교
방법DV 설정MDV장단점
M101 (제외)가장 흔하고 간단. BLQ 비율 낮을 때 적절. 높으면 bias.
M30 (censored)0 (포함)Likelihood-based. 통계적으로 가장 우수. Control stream 복잡. $ESTIMATION LAPLACIAN 또는 LIKELIHOOD 필요.
M5LLOQ/20M1보다 덜 편향되지만 informative한 것처럼 보이는 문제.
M7혼합혼합첫 BLQ만 M3, 이후 연속 BLQ는 M1. 균형 잡힌 접근.
데이터셋 구현
공통: BLQ flag 컬럼 (0/1) + LLOQ 값 컬럼을 반드시 생성. 어떤 M-method를 쓰든 원본 정보 보존.
M1이면: DV=0, MDV=1. NONMEM이 해당 행을 likelihood에서 제외.
M3이면: DV=0, MDV=0 유지. Control stream의 $ERROR에서 censoring 로직 구현 (IF(BLQ.EQ.1) YLO=0, YHI=LLOQ 등).
선택 가이드
BLQ 비율 <10% → M1으로 충분. 10-25% → M5 또는 M3 고려. >25% → M3 강력 권장. 최종 결정은 모델러/팀과 협의.
케이스별
Outlier, Predose, Steady-state, 기타 특수 처리
Outlier 처리
원칙 — 삭제하지 말고 FLAG: 의심 데이터를 데이터셋에서 삭제하지 않고 EXCL 컬럼 (0=포함, 1=제외, 이유코드 등)으로 표시. NONMEM에서 IGNORE=(EXCL.GT.0)으로 제외. 원본 보존 + sensitivity analysis 가능.
Outlier 판단 기준: ① conditional weighted residual (CWRES) > |5|, ② 약동학적 비현실성 (Cmax 전에 고농도, 생물학적 불가능 수치), ③ documented protocol deviation. 주관적 판단은 반드시 문서화.
Predose sample
Single dose study: 첫 투여 전 채혈 = baseline. DV=0 확인 (detectable이면 carry-over 또는 assay issue 검토).
Multiple dose study: predose = trough (Cmin). TIME은 이전 dose로부터의 경과 시간. 새 dosing cycle 시작 전 시점.
Steady-state (SS) 데이터
언제 사용: 이전 투여 이력이 불완전할 때. 예: 외래환자가 매일 복용 중이나 투여 기록이 clinic visit만 → SS=1, II=투여간격 (hr)으로 NONMEM에게 steady-state 가정을 알림.
데이터 구현: 해당 dosing record에 SS=1, II=24(매일 투여 시). NONMEM이 시뮬레이션으로 SS 도달 후 해당 dose 적용.
체중 기반 투여 (mg/kg dosing)
AMT 기록법: absolute dose (mg)로 기록. mg/kg 아님. 예: 10mg/kg × 70kg = AMT=700. 체중은 covariate (WT)로 별도 기록. NONMEM이 내부적으로 WT-normalized CL 등 계산.
Rich vs Sparse sampling
Rich sampling: 한 ID에 8-15+ 채혈 시점. 전형적 Phase I. Individual parameter 추정 가능.
Sparse sampling: 한 ID에 1-3 채혈. Phase II/III. Population approach의 핵심 강점이 발휘되는 데이터. 데이터 구조는 동일하되 TIME 정확성이 더욱 중요 (시점이 적으므로 각 시점의 가치 극대화).

Baseline covariates 처리
소스 & 추출
DM/ADSL 도메인에서: AGE, SEX, RACE, ETHNIC, COUNTRY. 변환 없이 직접 사용 가능 (숫자 코딩 확인).
VS 도메인에서: Baseline WT, HT, BMI. "baseline"의 정의 = 첫 투여 전 마지막 측정값 (screening 또는 Day 1 predose).
LB 도메인에서: Baseline CLCR (또는 eGFR), ALB, ALT, AST, BILI. 신기능/간기능 지표.
결측 처리 의사결정
결측률 <5%: Median imputation (연속형) 또는 mode imputation (범주형). 문서화 필수.
결측률 5-20%: Multiple imputation 고려. 또는 LOCF (Last Observation Carried Forward).
결측률 >20%: 해당 covariate를 모델에 포함하기 어려울 수 있음. 모델러와 논의.
NONMEM 데이터에서: 빈 셀 불가 → -99 또는 . 으로 코딩. Control stream에서 IF(COV.EQ.-99) COV=MEDIAN 등으로 처리.
범주형 변수 코딩
NONMEM은 숫자만 처리 → SEX: Male=0, Female=1 (또는 역순). RACE: 0/1/2 등. 코딩 convention을 data specification에 명시. 이진 코딩이 해석에 가장 편리.
파생 변수 계산
BSA: DuBois 공식: BSA = 0.007184 × WT0.425 × HT0.725. R에서 계산 후 컬럼 추가.
CLCR (Cockcroft-Gault): ((140-AGE) × WT) / (72 × SCR) × (0.85 if female). SCR = serum creatinine. LB에서 추출.
LBW (Lean Body Weight): 비만 환자 분석 시 활용. Janmahasatian 공식 등.
복잡도 높음
Time-varying covariates 처리
NONMEM의 처리 방식
NONMEM은 각 record의 시점에서 해당 row에 기록된 covariate 값을 사용. 다음 record까지 그 값을 유지 (step function). 따라서 covariate 변경 시점에 covariate-only record를 삽입하거나, observation/dose record에 해당 시점의 covariate 값을 직접 기록.
Merge 전략
LOCF (Last Observation Carried Forward): 가장 흔한 방법. 각 record의 TIME보다 이전의 가장 가까운 covariate 측정값을 사용. R에서 dplyr::fill() 또는 data.table::nafill() 활용.
Time-point matching: Covariate 측정 시점과 PK 채혈 시점이 같은 visit에 해당하면 직접 merge. 시점이 다르면 LOCF.
Interpolation: 선형 보간. 잘 사용하지 않지만 체중 변화가 점진적일 때 고려 가능.
실무 주의점
첫 투여 전 covariate 측정값이 없는 ID → baseline 결측. 이 ID의 모든 record에 covariate가 결측될 수 있음.
병용약물 (concomitant medication): on/off 이진 변수로 코딩. CM 시작일/종료일 → 해당 기간의 record에 COMED=1.
Renal/Hepatic function 변화: ICU 환자 등에서 급격히 변할 수 있으므로 time-varying 처리가 중요.

$INPUT 정합성
NONMEM control stream의 $INPUT 행에 나열된 변수 순서 = 데이터 파일의 컬럼 순서. 반드시 일치해야 함. 사용하지 않는 컬럼은 DROP 또는 SKIP으로 처리. 데이터 컬럼을 추가/삭제하면 $INPUT도 수정.
규정 위반 = 실행 불가
NONMEM 포맷 규칙
빈 셀 없음: 결측은 . (마침표) 사용. R의 NA를 .으로 변환 필수.
소수점: 반드시 . (콤마 안 됨). 유럽 locale 데이터 주의.
문자열 불가: 모든 값은 숫자 또는 .. header를 제외하고 알파벳 포함 불가.
Header row: 정확히 1줄. NONMEM이 첫 줄을 변수명으로 읽음.
구분자: 공백 또는 콤마 (.csv). 일관성 유지. 탭은 비권장.
파일 인코딩: ASCII 또는 UTF-8 (BOM 없이). Excel에서 저장 시 "CSV UTF-8" 형식 주의.
컬럼명: 최대 8자 권장 (구버전 NONMEM 호환). 특수문자 불가. 밑줄(_)은 사용 가능하지만 비권장.
행 수 제한: NONMEM 7.4+ 에서는 사실상 무제한이지만, 매우 큰 데이터셋(>100만 행)은 $SIZES에서 LIM 조정 필요할 수 있음.

시각적 QC (Visual Inspection)
필수 플롯 5종
① Individual PK-Time profiles: 각 ID별 DV vs TIME (linear + log scale). 투여 시점을 vertical line 또는 marker로 표시. dose-concentration 시간 정합성을 확인하는 가장 강력한 도구. 전체 ID를 다 봐야 함.
② Dose vs TIME overview: AMT vs TIME scatter (전체 ID). 용량 변경, 투여 중단, 누락 패턴을 한눈에 파악.
③ Covariate distributions: Histogram/boxplot으로 각 covariate의 분포, 이상치, 결측 패턴 확인. Covariate 간 상관관계 pairs plot도 유용.
④ BLQ 패턴: BLQ 비율 vs TIME (binned). Terminal phase에서 BLQ 증가 = 정상. 초기 시점 BLQ = 의심 (dosing error? assay issue?).
⑤ Spaghetti plot: 전체 ID overlay (linear + log). 전반적 PK 경향, 극단값(outlier ID), dose-proportionality 시각적 확인.
눈으로 잡아야 할 Red Flags
투여 직후에 농도가 0인 ID → dosing record 누락 또는 TIME 오류
농도가 시간에 따라 지속적으로 증가 (accumulation 아닌 상황에서) → TIME 계산 오류 (보통 reference time 문제)
같은 시점에 극단적으로 다른 농도 → 단위 오류 또는 데이터 입력 오류
Dose 직전에 비현실적으로 높은 trough → carry-over, 투여 기록 누락, 또는 TIME 오류
특정 site/center에서만 비정상 패턴 → site-specific 데이터 입력 오류 가능

NMdata workflow (Philip Delff 방식)
NMdata::NMcheckData(): 실행 전 구조적 체크 한 번에 수행. 모든 문제를 리포트로 출력.

#실수결과방지법
1TIME 계산에서 reference time 불일치PK 파라미터 전체 틀림reference time 로직을 R 함수로 정의하고 전체 데이터에 일괄 적용
2같은 TIME에 obs가 dose보다 먼저 정렬predose로 해석되어 예측값 이상정렬 시 EVID 우선순위 반영: arrange(ID, TIME, desc(EVID))
3AMT에 planned dose 사용 (actual 대신)CL/V 추정 편향EX 도메인의 actual dose 컬럼 확인. 부분 투여 기록 확인.
4NA를 .으로 미변환NONMEM 실행 오류R 저장 시 write.csv(..., na=".") 또는 NMwriteData() 사용
5단위 불일치 (mg vs μg, L vs mL)CL이 1000배 차이Data spec에 단위 명시. 변환 로직 스크립트에 기록.
6BLQ를 그냥 0으로 두고 MDV=0 (제외 안 함)terminal phase biasBLQ flag 생성 + M-method 명시적 적용
7ADDL/II에서 용량 변경 시점 무시NONMEM이 잘못된 용량으로 추가 투여용량 변경 시점에서 ADDL 끊고 새 dosing record 시작
8Covariate merge 후 row 수 변동데이터 증폭 또는 소실merge 전후 nrow() 비교. 1:1 merge 확인.
9$INPUT 순서와 데이터 컬럼 순서 불일치변수 값이 뒤바뀜 (silent error)control stream 작성 시 데이터 header와 1:1 대조
10Outlier를 행 삭제로 처리원본 복원 불가, sensitivity analysis 불가EXCL flag 사용. IGNORE statement로 제외.
