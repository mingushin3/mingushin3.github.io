네 가지 질문에 먼저 직답하고, 그 답들을 묶어 v4로 정렬합니다.

## 네 직답

**Q1 — Rescue 100 독립 마련 vs 2600에서 추출?** **독립 마련.** 두 corpus는 선택 함수가 다릅니다. 2600은 "고품질 화용론적 brilliance" 기준이고, Rescue는 "freeze 상태에서 0.5초 자동 인출" 기준입니다. "Sorry, I missed that" 같은 회복 문장은 brilliance가 부족해 페르소나 프롬프트의 "태움" 판정을 못 받습니다 — 하지만 그게 정확히 Rescue의 본질입니다. 2600 안에서 Rescue를 찾는 작업은 잘못된 어장에서 다른 종을 찾는 일입니다. ChatGPT + Claude + Gemini 교차검증으로 이번 주 확정 — 진행 권장.

**Q2 — 100 vs 300?** **100이 정답.** 300이 아닌 이유: 0.5초 자동 인출은 과학습(overlearning)을 요구하고, 과학습은 작은 set에만 가능합니다. 300이면 분산되어 어느 것도 자동화 안 됩니다. 100은 적은 게 아니라 **forcing function** — 강제 우선순위 메커니즘입니다. Phase 1에서 100 완전 자동화 → Phase 2에서 Priority A 300으로 자연스럽게 확장. 순서가 정해져 있습니다.

**Q3 — 800 의미 부여 vs 2600 자체 가치?** 당신이 맞습니다. 800은 1회독에 불과하므로 sunk cost. **2600 자체는 강력 자산이지만 1년 안에 모두 처리한다는 사고는 폐기.** v3의 stratification 그대로.

**Q4 — YouTube 즉각 피드백 vs 프랙탈 함정?** **YouTube에서 피드백 루프를 분리합니다.** 모든 피드백은 morning Voice LLM spar에서 완결. YouTube는 다운스트림 artifact일 뿐 학습 입력 아님. 자막 audit, 재촬영, 재업로드 사이클 = 폐기. **이게 v4의 핵심 변화입니다.**

## 2600 텍스트화 한 달이라는 가정 — 사실 한 달 아닙니다

당신이 한 달이라고 추정한 이유는 수동 타이핑을 가정했기 때문입니다. 실제로는 두 가지 자동화 경로가 있습니다.

**경로 A**: 페르소나 프롬프트 입력 history에서 텍스트 추출. 당신이 "태움" 판정을 받기 위해 입력했을 때 이미 텍스트로 들어갔습니다. ChatGPT/Claude/Gemini 채팅 기록에 텍스트 형태로 존재합니다. 추출만 하면 됩니다 — 1-2시간.

**경로 B**: history가 분산되어 있다면 Whisper API로 2600 영상 클립을 batch 전사. 한 클립 평균 5초 × 2600 = 13,000초 = 3.6시간 분량. Whisper는 5-10x 실시간 처리 = API 콜 30-45분. 비용 약 1.30달러.

어느 경로든 **주말 한 번이면 끝납니다.** 그 다음 LLM batch tier 분류가 2-3시간. **총 5-8시간으로 종결**됩니다. 한 달이라는 추정 자체가 v4가 깨야 할 첫 가정입니다.

## v3 → v4 정확한 diff

**변경 1 — Voice LLM Spar가 모든 피드백의 단일 채널.** 텍스트 spar는 보조 수단. ChatGPT Plus의 Voice mode + Gemini Live가 메인. 매일 아침 15분이 유일한 정규 피드백 시간입니다.

**변경 2 — YouTube 피드백 루프 완전 제거.** 자막 audit, paradigm shift 추출, 재촬영 모두 폐기. YouTube = 1테이크 30초 NG 그대로 업로드 + 끝. 일일 5분.

**변경 3 — 2600 처리 일정 현실화.** 한 달 → 주말 1회. LLM batch 자동화로 압축.

**변경 4 — Rescue 100 = 독립 데이터셋.** 2600에서 추출 시도 금지.

**변경 5 — Sunday Diagnostic 간소화.** 7개 지표 → 핵심 3개 (response latency · breakdown count · repair skill)로. 측정 시간 20분 → 10분.

## v4 데일리 플로우**아침 15분 — Voice LLM Spar (유일한 정규 피드백 채널)**: ChatGPT Voice 또는 Gemini Live. 5개 질문 방어. 막힌 순간 LLM이 즉시 음성으로 paradigm shift sentence 제공. **이게 v4의 단일 피드백 시간입니다.** 다른 어디에도 audit 루프 없음.

**출근 30분 — Shadowing**: 2600 archive 영상 클립 10개. 3-stage (Echo / Delayed / No-audio recall).

**점심 10분 — Rescue Drill A**: 100문장 중 10개 SRS recall. 한국어 의도 → 영어 즉답 0.5초.

**퇴근 30분 — Variation Drill**: 아침 spar에서 들었던 paradigm shift sentence + skeleton을 4 도메인 (계량약리학·비즈니스·친목·일상)으로 즉흥 변형. 자료 없이 입으로만.

**저녁 5분 — YouTube Upload**: 1테이크 30초 NG 그대로. 자막 audit 안 함. 공개 commitment + 1년 누적 archive 기능만 수행.

**자기 전 5분 — Rescue Drill B**: 다른 5문장 SRS recall. 100문장 = 매주 1.05회독.

총 **새 시간 35분 + 출퇴근 60분**. 지속 가능합니다.

## v4 Week 1 sprint

**Day 1 (월) — Rescue 100 시드 확정 (3시간)**

3개 LLM에 동일 프롬프트:
```
5개 카테고리 × 20문장 = 100문장 생성.
카테고리: (1) 못 알아들었을 때, (2) 생각할 시간 필요할 때, 
(3) 말 다시 잡을 때, (4) 부드럽게 반대할 때, (5) 친근하게 반응할 때.
조건: 한국인 성인이 freeze 상황에서 0.5초 안에 자동 인출되어야 할 
formulaic phrases. COCA/iWeb 코퍼스 검증 우선.
```
3 LLM 출력 비교 → 합의 70개 + 페르소나 프롬프트 validate된 30개 = 100문장 lock.

**Day 2 — TTS 일괄 생성 (30분)**

ElevenLabs 무료 티어 또는 OpenAI TTS API. Python 스크립트 30줄. 100 mp3 자동 생성. 비용 5-10달러.

**Day 3 — Anki/Obsidian SRS 등록 (1시간)**

Front: 한국어 의도 + 1줄 상황. Back: 영어 + audio.

**Day 4 — Voice Spar 셋업 (1시간)**

ChatGPT Plus의 Custom GPT 만들기 → system prompt에 v3 Asset 3 spar 프롬프트 그대로 입력. Voice mode 진입. 첫 spar 15분 시범. (Claude의 voice 도구가 출시되면 즉시 전환.)

**Day 5 — YouTube 채널 셋업 (30분) + 첫 영상 업로드**

자동 자막 ON. 첫 영상은 "Why I'm doing this for 365 days" 메타 commitment. 한국어 자막 병행 권장.

**Day 6 (토) — 2600 텍스트 추출 + LLM batch 분류 (3-4시간)**

페르소나 프롬프트 history에서 텍스트 추출 → 100문장 batch × 26회 LLM 분류 → Priority A/B/C 자동 산출. Priority A 300개 식별이 목표. B/C는 분류만 하고 깊이 처리 안 함.

**Day 7 (일) — 인간 채널 lock-in (30분)**

iTalki 또는 Cambly에서 학술 영어 가능 튜터 1명 선정 + 첫 세션 일주일 후 예약. 추가로 PIPET 랩 국제 방문자 / 영어 저널 클럽 가능성 탐색.

7일 끝 = 시스템 가동.

## 4개 layer가 서로 간섭 없이 돌아간다

v4의 본질은 단순합니다.

**Layer 1 — Rescue 100 (단권화)**: TTS + SRS, 매일 15문장, 1년 200-300회독. 자동화 목표.

**Layer 2 — 2600 archive (인풋 reservoir)**: 영상 클립 stratified, Priority A 300 깊이 처리, 나머지는 reference.

**Layer 3 — Voice spar (피드백)**: 매일 아침 15분, 실시간 음성 교정. 모든 학습 피드백의 단일 채널.

**Layer 4 — YouTube + 인간 대화 (출력 압력)**: YouTube 일일 30초 (공개 archive) + 주 1회 인간 대화 (정서적 활성화).

각 layer가 자기 함수만 수행하고 서로 간섭하지 않습니다. 프랙탈 함정은 layer 간 cross-feedback 시도에서 생기는데, v4에서는 그런 시도 자체가 없습니다.

## 한 문장

**즉시 피드백은 voice spar에서 종결, YouTube는 다운스트림 archive, 단권화는 Rescue 100 TTS, reservoir는 stratified 2600 — 4개 layer가 cross-feedback 없이 각자 돌면 1년이 너무 깁니다.** 첫 voice spar는 내일 아침, 첫 YouTube는 오늘 저녁. 둘 사이에 audit 활동 없음.
