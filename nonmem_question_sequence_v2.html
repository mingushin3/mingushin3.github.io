Q01–Q06
Q01 데이터 형태 식별 & 파일 인벤토리
MUST
업로드한 데이터 파일들을 확인하고, (1) 파일 형식 (xpt/csv/xlsx/sas7bdat), (2) 데이터 유형 (SDTM/ADaM/Raw CRF/Excel/기존 NONMEM), (3) 각 파일의 행·열 수와 변수 목록을 표로 정리해줘. 파일이 여러 개면 각각 정리하고, SDTM이면 도메인명(DM, EX, PC 등)도 식별해줘.
🎯 LLM 출력물
① 파일 인벤토리 테이블 (파일명 / 형식 / 유형 / 행수 / 열수 / 핵심 변수)
② 데이터 유형 판정 결과 (SDTM/ADaM/Raw/Excel/기존NM)
③ 후속 질문 중 건너뛸 수 있는 COND 질문 사전 식별
파일 확장자로 형식 판별 → haven::read_xpt(), readr::read_csv(), readxl::read_excel() 등 적절한 reader 선택. SDTM이면 DOMAIN 변수 또는 파일명에서 도메인 식별. ADaM이면 ADSL/ADPC/ADEX 존재 확인.
→ Phase 0: 전체 소스 유형 분기
Q02 Subject ID 식별 & 고유 대상자 수 확인
MUST
데이터에서 대상자 식별 변수(USUBJID, SUBJID, PTNO 등)를 찾고, 고유 대상자 수를 세어줘. ID 형식이 문자열이면 숫자 ID로 재코딩하는 매핑 테이블도 만들어줘. 여러 파일에서 ID 교집합/차집합도 확인해줘.
🎯 LLM 출력물
① ID 매핑 테이블 (원본ID → 숫자ID)
② 파일 간 ID Venn diagram 정보 (교집합/차집합 수)
③ R 코드: id_map <- dm %>% distinct(USUBJID) %>% mutate(ID = row_number())
→ Phase 0: Subject ID 체계 통일
Q03 날짜/시간 변수 식별 & 형식 평가
MUST
모든 파일에서 날짜/시간 관련 변수를 찾고, 각각의 형식(ISO 8601? AM/PM? 날짜만?), 결측률, 파싱 가능 여부를 표로 정리해줘. 투여 시점과 채혈 시점의 시간 정밀도(분 단위까지 있는지)를 확인해줘.
🎯 LLM 출력물
① 날짜/시간 변수 인벤토리 테이블 (변수명 / 형식 / 정밀도 / 결측률 / 파싱가능여부)
② 날짜+시간 분리 여부 판정
③ AM/PM, locale 혼재 시 변환 전략 제안
→ Phase 2 선행 조건
Q04 핵심 소스 테이블 매핑
MUST
NONMEM building block 3개의 소스를 매핑해줘:
• Dosing 소스: 어떤 파일/테이블의 어떤 변수가 투여 기록인가?
• Observation 소스: PK 농도(DV)는 어디에? 단위는? BLQ 표시는?
• Covariate 소스: demographics, lab, vitals는 어디에?
각각 [소스파일 → 핵심변수 → NONMEM 변수] 매핑 테이블을 만들어줘.
🎯 LLM 출력물
① 3-block 매핑 테이블 (소스 → NONMEM 변수)
② DDT (Data Definition Table) 초안 (Raw CRF/Excel인 경우)
③ 누락된 핵심 변수 경고 (있다면)
→ Phase 0: DDT 수립 / 6대 표준 입력 확인
Q05 기존 NONMEM 데이터셋 인수인계 체크
COND기존NM만
이 데이터셋은 기존 NONMEM 데이터셋이다. (1) TIME 기준점을 역추적해줘. (2) FLAG/IGNORE/EXCL 컬럼 convention을 파악해줘. (3) $INPUT 변수 순서를 데이터 컬럼 순서와 대조해줘. (4) 새 데이터 추가 시 ID 충돌 방지 방안도 제안해줘.
🎯 LLM 출력물
① TIME reference 역추적 결과
② 기존 $INPUT 문자열 vs 데이터 컬럼 대조표
③ FLAG/EXCL convention 요약
④ ID 충돌 방지 전략 (offset 등)
⏭ 기존 NONMEM 데이터를 수정/확장하는 경우에만 해당. 새로 만드는 경우 건너뛰기.
→ Phase 0: 기존 데이터셋 수정/확장
Q06 Multi-study pooling 전략 수립
CONDNEW다수study
2개 이상 study를 pooling하는 경우: (1) STUDYID 컬럼을 생성하고 숫자 코딩해줘. (2) study 간 AMT/DV 단위 일치 여부를 확인해줘. (3) bioanalytical assay가 다르면 LLOQ 차이를 보고해줘. (4) ID가 study 간 충돌하지 않도록 offset을 적용해줘 (예: Study B의 ID = Study A max ID + 100).
🎯 LLM 출력물
① STUDYID 매핑 테이블 (study명 → 숫자코드)
② 단위 일치 확인 결과 (AMT, DV, TIME)
③ Assay 차이 보고 (LLOQ, 분석법)
④ ID offset 적용 R 코드
⑤ Study별 대상자수/투여횟수 요약
⏭ 단일 study면 건너뛰기. STUDYID=1로 고정하고 진행.
→ Phase 0: Multi-study pooling / 기존 데이터 확장

Dosing Records 조립

Q07–Q18
Q07 투여 기록 추출 & 기본 구조 생성
MUST
dosing 소스에서 투여 기록을 추출해서 다음 컬럼을 가진 dosing 데이터프레임을 만들어줘: ID, DATETIME, AMT, CMT, EVID=1, MDV=1, DV=0. 실제 투여 여부 확인: EXSTAT="NOT DONE"인 건 제외. 각 ID별 투여 횟수, 총 투여 record 수를 요약해줘.
🎯 LLM 출력물
① dosing 데이터프레임 구조 확인 (head(), str())
② ID별 투여 횟수 요약 (min/median/max)
③ 제외된 "NOT DONE" record 수 보고
④ R 코드 전문
→ Phase 1: Dosing records 필수 변수
Q08 Actual vs Planned dose 확인
MUST
AMT에 들어간 값이 실제 투여량(actual dose)인지 계획 투여량(planned dose)인지 확인해줘. Actual과 planned가 다른 record를 찾아 목록으로 보여줘. AMT는 반드시 actual dose여야 하고, planned dose는 별도 PDOSE 컬럼으로 보존해줘.
🎯 LLM 출력물
① Actual ≠ Planned 불일치 record 목록 (ID, TIME, actual, planned)
② 불일치 비율 (%) 및 원인 추정 (부분주입, 구토, dose reduction 등)
③ PDOSE 컬럼 생성 R 코드
→ Phase 1: Actual vs Planned dose
Q09 투여 경로 & RATE/DUR 설정
MUST
투여 경로(PO/IV bolus/IV infusion/SC/IM)를 확인하고, 경로에 맞게 RATE와 DUR을 설정해줘:
• 경구/SC/IM bolus → RATE=0
• IV bolus → RATE=0
• IV infusion → RATE=-2 + DUR=주입시간(hr) (또는 RATE=AMT/DUR)
혼합 경로이면 각각 다른 CMT 할당 고려.
🎯 LLM 출력물
① 투여경로 분포 테이블 (경로 / record수 / %)
② RATE, DUR 컬럼 설정 R 코드
③ IV infusion의 DUR 계산 로직 (EXSTDTC~EXENDTC 차이 등)
→ Phase 1: 투여 경로별 의사결정 트리
Q10 AMT 단위 확정 & 변환
MUST
AMT의 단위를 확인하고 확정해줘(mg? μg? cells?). DV 단위와의 조합이 PK 파라미터 단위를 결정하므로, AMT(단위) ÷ DV(단위) → CL(단위) 관계를 명시해줘. 단위 변환이 필요하면 수행하고 변환 로직을 기록해줘.
🎯 LLM 출력물
① 단위 관계식: AMT(___) / DV(___) → CL(___), V(___)
② 단위 변환 R 코드 (필요시)
③ AMT 범위 요약 (min, median, max, unique values)
→ Phase 1: 단위
Q11 체중 기반 투여 → absolute dose 변환
CONDNEWmg/kg투여
투여가 mg/kg 기반이면: AMT를 absolute dose(mg)로 변환해줘. 예: 10mg/kg × 70kg = AMT=700. 계산에 사용된 체중은 WT 컬럼으로 별도 보존. 변환 전후 AMT를 대조하여 검증해줘.
🎯 LLM 출력물
① 변환 R 코드: mutate(AMT = AMT_MGKG * WT)
② 변환 전후 AMT 비교 요약 (min/max 변화)
③ WT 결측인 ID 경고 (AMT 계산 불가)
⏭ Flat dose (mg 기반) 투여이면 건너뛰기.
→ Phase 3: 체중 기반 투여
Q12 CMT 할당 (Dosing compartment)
MUST
dosing record의 CMT 번호를 할당해줘. 경구 → CMT=1 (depot), IV → CMT=2 (central). 혼합 경로이면 경로별 CMT 매핑을 명시해줘.
🎯 LLM 출력물
① CMT 매핑 규칙 테이블 (경로 → CMT)
② CMT 할당 R 코드
③ CMT별 record 수 분포
→ Phase 1: CMT 설정
Q13 반복 투여 압축 (ADDL/II)
COND반복투여
동일 용량의 반복 투여 구간을 찾아서 ADDL과 II(hr)로 압축해줘. 용량 변경 시점에서는 ADDL을 끊고 새 dosing record를 시작. 압축 전후 총 투여 횟수가 동일한지 검증해줘.
🎯 LLM 출력물
① 압축 전 총 투여 record 수 vs 압축 후 record 수
② 총 투여 횟수 동일성 검증 결과
③ 용량 변경 시점 식별 목록
④ R 코드 전문
⏭ Single dose study이면 건너뛰기.
→ Phase 1: 반복 투여 패턴 / ADDL/II
Q14 투여 중단/재시작 & Reset (EVID=4)
COND
투여 중단 후 재시작이 있는 대상자가 있는지 확인해줘. Washout 후 재투여면 EVID=4(reset+dose)가 필요한지 판단. 중단 기간을 식별하고, 재시작 시 reset 적용 여부를 결정해줘.
🎯 LLM 출력물
① 투여 gap이 큰 ID 목록 (ID, gap 기간)
② EVID=4 적용 판정 결과
③ EVID 수정 R 코드
⏭ 연속 투여이고 중단/재시작 없으면 건너뛰기.
→ Phase 1: Reset (EVID=4)
Q15 Steady-state 처리 (SS/II)
COND
이전 투여 이력이 불완전한 대상자가 있는지 확인해줘 (외래환자 매일 복용 중이나 clinic visit 기록만 존재). 해당되면 SS=1, II=투여간격(hr)을 설정하는 dosing record를 만들어줘.
🎯 LLM 출력물
① SS 적용 대상 ID 목록 및 근거
② SS/II 설정 R 코드
③ SS record 검증 (SS=1이면 II>0 확인)
⏭ 모든 투여가 기록된 경우 건너뛰기.
→ Phase 3: Steady-state
Q16 특수 투여: CAR-T / 세포치료제 / 생물제제
COND해당시만
세포치료제/CAR-T이면: AMT=주입 세포수(single infusion), 이후 세포 expansion은 DV(EVID=0)로 처리. DV가 투여 후 증가하는 패턴이 정상임을 인지. dosing record를 1회 infusion으로 설정해줘.
🎯 LLM 출력물
① Dosing record 구조 (AMT=세포수, EVID=1, 단위 명시)
② DV 패턴 확인 (expansion→contraction 정상 확인)
⏭ 일반 소분자/항체 약물이면 건너뛰기.
→ Phase 1: CAR-T 특수 사례
Q17 EVID=2 기타 이벤트 처리
CONDNEWturnover등
모델에 EVID=2 (Other type event)가 필요한 경우: turnover model의 외부 input, 자극/억제 이벤트, 식사 시점 기록 등. 해당 이벤트를 식별하고 EVID=2, MDV=1, AMT=0 record를 생성해줘. 이벤트의 의미와 시점을 명시.
🎯 LLM 출력물
① EVID=2 이벤트 정의 및 소스 식별
② EVID=2 record 생성 R 코드
③ 생성된 EVID=2 record 수 및 분포
⏭ 표준 PK 모델(EVID=0,1만 사용)이면 건너뛰기.
→ Phase 5: EVID 조합 패턴 — EVID=2 행
Q18 Dosing 데이터 중간 검증
MUSTAUTO
dosing 데이터프레임을 검증해줘:
✓ 모든 행에 EVID=1(또는 4), MDV=1인가?
✓ AMT > 0인가?
✓ 모든 ID에 최소 1개 dosing record 존재?
✓ DATETIME에 결측이 있는가?
✓ ID별 투여 횟수 요약 통계
문제가 있으면 수정 코드도 함께 제공.
🎯 LLM 출력물
① 5개 체크 항목 PASS/FAIL 리포트
② 실패 항목의 문제 record 목록
③ 자동 수정 R 코드 (필요시)
→ Phase 6: Programmatic QC (dosing)
1b
Observation Records 조립
EVID=0 — PK/PD 관찰값 구성
Q19–Q25
Q19 PK 관찰값 추출 & DV 설정
MUST
observation 소스에서 PK 농도 데이터를 추출하여 ID, DATETIME, DV, CMT, EVID=0, MDV, AMT=0 구조의 데이터프레임을 만들어줘. DV 단위를 확정하고, PCSTRESN과 PCORRES가 다른 행을 보여줘.
🎯 LLM 출력물

Q49 필수 컬럼 존재 확인 & 순서 배치
MUSTAUTO
다음 필수 컬럼이 모두 존재하는지 확인하고, $INPUT 호환 순서로 배치:
절대 필수: ID, TIME, DV, AMT, CMT, EVID, MDV
거의 필수: RATE, ADDL, II, SS, BLQ, LLOQ (해당시)
파생: TAFD, TAD
공변량: WT, AGE, SEX, CRCL, ...
관리: EXCL, STUDYID, NMSEQNO, DVID (해당시)
🎯 LLM 출력물
① 컬럼 존재 체크 PASS/FAIL
② 누락 컬럼 자동 생성 (기본값)
③ 최종 컬럼 순서 및 select R 코드
→ Phase 5: 필수 컬럼 & 정렬 규칙
Q50 최종 정렬: ID → TIME → EVID 우선순위
MUST
최종 정렬: 1차 ID ↑ → 2차 TIME ↑ → 3차 EVID 내림차순 (4>1>2>0). NMSEQNO 행번호 추가.
🎯 LLM 출력물
① 정렬 + NMSEQNO R 코드
② 정렬 후 ID 1~2명 head 출력으로 검증
→ Phase 5: 정렬 규칙
Q51 NA → "." 변환 & 문자열 제거
MUST
(1) R의 NA를 모두 .으로 변환. (2) header 제외 모든 값이 숫자 또는 .인지 확인. (3) 유럽 locale 콤마 소수점 주의.
🎯 LLM 출력물
① NA 변환 R 코드
② 문자열 잔류 컬럼/행 목록 (있다면)
③ 소수점 형식 확인 결과
→ Phase 5: NONMEM 포맷 규칙
Q52 Non-ASCII & 파일 인코딩 검증
MUSTNEWAUTO
데이터에 non-ASCII 문자(한글, 특수기호, BOM 등)가 있는지 검사해줘. NONMEM은 ASCII만 안정적으로 처리. non-ASCII가 있으면 해당 행/컬럼을 보고하고 제거 또는 변환해줘. 파일 인코딩을 UTF-8 (BOM 없이)로 확정.
🎯 LLM 출력물
① Non-ASCII 검출 R 코드
② 검출된 non-ASCII 문자 목록 (행/컬럼/값)
③ 정리 R 코드 + 인코딩 확인
→ Phase 5: 파일 인코딩 / Phase 6: non-ASCII 체크
Q53 컬럼명 규격 확인
MUSTAUTO
컬럼명 확인: (1) 8자 이내? (2) 특수문자 없음? (3) 대문자 표기? 위반 컬럼은 rename.
🎯 LLM 출력물
① 위반 컬럼 목록 및 rename 매핑
② rename R 코드
③ 최종 컬럼명 목록
→ Phase 5: 컬럼명 규칙
Q54 CSV 파일 저장
MUST
최종 CSV 저장: write.csv(nm_df, "nm_data.csv", row.names=FALSE, na=".", quote=FALSE). 또는 NMdata::NMwriteData() 사용. 저장 후 파일을 다시 읽어 원본과 일치 확인.
🎯 LLM 출력물
① 저장 R 코드
② Round-trip 검증: 다시 읽어서 dim/head 비교
③ 파일 크기, 행수, 열수 최종 보고
→ Phase 5: 파일 포맷
Q55 $INPUT 문자열 생성
MUSTAUTO
NONMEM $INPUT 행을 자동 생성. 데이터 컬럼 순서와 정확히 일치. 사용하지 않는 컬럼은 DROP. 동시에 $DATA 행도 생성 (파일명, IGNORE 조건).
🎯 LLM 출력물
① $INPUT 문자열 (복사 가능)
② $DATA 문자열 (파일명 + IGNORE 조건)
③ 컬럼 순서 vs $INPUT 대조 확인
→ Phase 5: $INPUT 정합성

Q56–Q62
Q56 구조적 무결성 종합 체크
MUSTAUTO
최종 데이터셋에 대해 전체 구조적 QC를 실행:
✓ 모든 ID에 ≥1 dosing record?
✓ 모든 ID에 ≥1 valid observation?
✓ EVID=1 → AMT>0, MDV=1?
✓ EVID=0 → AMT=0 or .?
✓ 중복 행 없음? (ID+TIME+EVID+CMT)
✓ CMT 유효 범위?
✓ ADDL/II 사용 시 II>0, EVID=1 or 4?
✓ SS=1 → II 존재?
✓ 데이터에 NA/non-numeric 없음?
✓ Header 정확히 1줄?
✓ 각 ID의 첫 record가 dosing record인지? (아니면 의도적인지 확인)
✓ non-ASCII 문자 없음?
PASS/FAIL 항목별 보고.
🎯 LLM 출력물
① 12개 체크 항목 PASS/FAIL 리포트 테이블
② FAIL 항목의 문제 record 목록
③ NMdata::NMcheckData() 실행 결과 (사용 가능 시)
④ 자동 수정 가능 항목의 R 코드
→ Phase 6: Programmatic QC 전체
Q57 EVID/MDV 조합 Cross-tabulation
MUSTAUTO
EVID × MDV × BLQ(해당 시) 교차표를 만들어줘. 각 조합의 행 수를 세고, 비정상 조합(예: EVID=1이면서 MDV=0, EVID=0이면서 AMT>0)이 있으면 경고.
🎯 LLM 출력물
① EVID×MDV×BLQ 교차표
② 비정상 조합 경고 목록
③ R 코드: nm_df %>% count(EVID, MDV, BLQ)
→ Phase 5: EVID·MDV 조합 패턴
Q58 Individual PK-Time Profile 플롯
MUSTVISUAL
모든 ID에 대해 DV vs TIME 개인별 프로파일 (linear + log scale). 투여 시점을 세로선/화살표로 표시. facet_wrap으로 ID별 패널. Red flags:
🚩 투여 직후 농도=0
🚩 지속 증가 (비축적 상황)
🚩 극단 농도값
🎯 LLM 출력물
① ggplot2 R 코드 (linear + log, facet_wrap)
② 플롯 출력 (또는 PDF 저장)
③ Red flag 해당 ID 목록 (있다면)
→ Phase 6: 시각적 QC ①② Individual profiles
Q59 Dose-TIME overview 플롯
MUSTNEWVISUAL
AMT vs TIME scatter plot (전체 ID overlay). X축=TIME, Y축=AMT, 색상=dose group 또는 ID. 이 플롯으로 용량 변경 패턴, 투여 중단, 누락을 한눈에 조감. site별 색상도 고려 (비정상 site 패턴 확인).
🎯 LLM 출력물
① Dose-TIME overview ggplot2 R 코드 + 플롯
② 용량 변경 시점 식별
③ 투여 gap/누락 패턴 보고
④ Site별 이상 패턴 보고 (multi-site인 경우)
→ Phase 6: 시각적 QC ② Dose vs TIME overview
Q60 Spaghetti Plot & Covariate 분포
MUSTVISUAL
(1) Spaghetti plot: 전체 ID overlay (DV vs TIME, linear + log).
(2) Covariate 분포: 연속형은 histogram/boxplot, 범주형은 bar chart.
(3) Covariate 상관관계: pairs plot (WT-HT, AGE-CLCR 등).
🎯 LLM 출력물
① Spaghetti plot R 코드 + 플롯
② Covariate 분포 플롯
③ 상관관계 pairs plot
④ 이상치 ID 또는 이상 패턴 보고
→ Phase 6: 시각적 QC ③⑤ Spaghetti + Covariate
Q61 Data Specification 문서 생성
MUST
Data Specification 문서를 생성해줘. 각 컬럼에 대해:
• 컬럼명 / 설명 / 단위 / 데이터 타입 / 결측 코딩 / 값 범위
• 코딩 규칙 (SEX: 0=M, 1=F 등)
• BLQ 처리 방법 / AMT 단위 / DV 단위 / TIME 기준점
• 제외 기준 요약 (몇 명/몇 record, 이유)
• 총 대상자 수, 총 record 수, dosing/obs record 수
🎯 LLM 출력물
① Data Specification 테이블 (컬럼별 완전 정의)
② 데이터셋 요약 통계 (N subjects, N records, N dose/obs)
③ 제외 기준 요약 (flowchart 형태 권장)
④ M-method, TIME reference, 단위 등 핵심 convention 명시
→ Phase 6: Data Specification Document
