CARD 01 NONMEM 3요소 + 제어구문 레코드 아키텍처 + $DATA 데이터 통제
개념 Big Idea: 거장의 시각

왜 치명적으로 중요한가: $DATA의 IGNORE 조건이 암묵적 OR로 작동한다는 사실을 모르면 AND 배제를 의도한 코드가 전혀 다른 부분집합을 분석하고도 ICH E9(R1) 수준 재현성 검증에서 발견되지 않는다.

체화해야 할 직관: 제어구문은 NONMEM에게 보내는 "법률 계약서"다 — $DATA·$PK·$THETA/$OMEGA·$ESTIMATION·$TABLE의 조항 하나가 모호하면 결과 전체의 효력이 무너진다.

이 직관을 박고 읽어라: 레코드 순서는 NM-TRAN 파싱 의존성 사슬이며, 이 불변성이 왜 필연인지를 이해하면 오류 메시지 없이도 구조적 결함을 예방할 수 있다.

A. 정의

NM-TRAN: 제어구문 파싱·문법검사 후 NONMEM 실행 가능 형태로 변환하는 번역기. PREDPP: PK 구획 모델 FORTRAN 서브루틴 라이브러리. NONMEM 엔진: OFV 최소화를 통한 파라미터 추정 알고리즘. 배치 모드(GUI 없음, ASCII 텍스트 제어구문만). [Owen, Ch.3, pp.28-29]

💡배치 모드란? Excel처럼 실시간 클릭이 아니라 제어구문 파일 하나를 던지면 처음부터 끝까지 자동 실행하는 방식이다. 코드 한 글자의 오류가 수 시간짜리 실행을 멈춘다 — 방어적 코딩이 선택이 아닌 필수인 이유다.

B. 제어구문 전체 구조 (파싱 의존성 사슬)

; ① 프로젝트 식별자
$PROBLEM 1-compartment oral PK model
; ② 데이터셋 열 선언 (최대 50개)
$INPUT ID TIME AMT RATE DUR DV MDV WT AGE SEX CLCR
; ③ 파일 경로 + 부분집합화
$DATA dataset.csv IGNORE=#
; ④ 구획 구조 + 파라미터화 선택
$SUBROUTINE ADVAN4 TRANS4
; ⑤ 고정효과 + IIV 정의
$PK
; ⑥ 잔차 구조
$ERROR
; ⑦ 초기추정값 (분산값 입력 — SD 아님!)
$THETA / $OMEGA / $SIGMA
; ⑧ 추정 알고리즘
$ESTIMATION METHOD=1 INTER MAXEVAL=9999
; ⑨ 표준오차 + 고유값
$COVARIANCE PRINT=E
; ⑩ 출력 파일 제어
$TABLE ID TIME DV IPRED CWRES ONEHEADER NOPRINT FILE=sdtab001

[출처: PIPET, Ch.3, pp.27-30] [출처: Owen, Ch.3, pp.30-31]

💡레코드 순서 불변 이유: NM-TRAN은 위에서 아래로 순차 파싱한다. $PK에서 WT를 참조하려면 $INPUT에 먼저 선언되어야 한다. 마치 법원 판결문에서 "제3조에서 정의된 A에 따라 제7조를 적용한다"고 쓰면, 제3조가 앞에 있어야 제7조가 효력을 갖는 것과 같다.

⚠ PIPET vs Owen 불일치 — IGNORE 다중 조건 AND/OR

PIPET p.31: ".AND. 또는 .OR. 사용 가능"이라고 기술 — 부정확. Owen p.33 "No other operator may be specified"가 엔진 동작의 정확한 기술이다.
; ✅ 올바른 이해 — 다중 조건은 암묵적 OR
$DATA test.csv IGNORE=(WTKG.GE.100, STDY.EQ.301)
; → "체중 ≥ 100kg OR 스터디 = 301"인 행 제외

; ✅ AND 배제 — ACCEPT 역전환 (De Morgan 법칙)
$DATA test.csv ACCEPT=(WTKG.LT.100, STDY.NE.301)
; NOT(A AND B) = NOT A OR NOT B — 방향 반전 + 부호 반전

; ✅ CHECKOUT: 추정 없이 파싱만 검증
$DATA test.csv IGNORE=# CHECKOUT

[출처: Owen, Ch.3, pp.33-35]

📌De Morgan 변환 직관: "A이면서 B인 사람 제외" → ACCEPT로 방향 뒤집고 부호 반전: "A가 아니거나 B가 아닌 사람만 포함". IGNORE → ACCEPT, 조건 부호 반전이 핵심.
💡CHECKOUT 실용 가치: 300명 데이터셋에서 IGNORE 조건 동작 확인 시, 수 시간 추정을 기다릴 필요 없이 수십 초 만에 $TABLE 출력 후 R에서 nrow()로 포함 행 수 검증 가능.

CARD 02 특수 ADVAN + TRANS 조합 — 구획 설계도·파라미터 언어·척도 통일 ⚡ Apex Concept
개념 Big Idea: 거장의 시각

왜 치명적으로 중요한가: IV 2구획(ADVAN3)을 경구 2구획(ADVAN4)으로 전환할 때 S-parameter와 CMT를 동시에 수정하지 않으면, NONMEM은 오류 메시지 없이 V2/1000배 낮은 농도를 예측하고 $COV step까지 성공시켜 IND 제출 직전까지 발견되지 않는다.

체화해야 할 직관: ADVAN = "구획 설계도", TRANS = "파라미터 언어", S-parameter = "스케일 변환 법칙". 설계도가 바뀌면 구획 번호 체계 전체가 재정의된다.

이 직관을 박고 읽어라: 홀수 ADVAN(1,3,11) = IV → 중심 CMT1 → S1. 짝수(2,4,12) = 경구 → Depot CMT1, 중심 CMT2 → S2.

📌홀짝 암기법: 홀수(1,3,11) = 주사바늘이 혈관에 바로 꽂힌다(IV) → 중심구획이 첫 번째(CMT1, S1). 짝수(2,4,12) = 입으로 삼킨다(경구) → 위장(Depot)이 먼저(CMT1), 혈관은 두 번째(CMT2, S2). 홀수=S1, 짝수=S2.

B. 특수 ADVAN 선택 체계 + CMT 번호 체계 (Owen 교정판)

PIPET Table 4.5 (p.53): ADVAN3 CMT를 "1=Depot"으로 표기 — 오기. IV 2구획에 Depot은 없다. Owen Table 3.2 기준을 따를 것.
ADVANCMT1CMT2CMT3관찰 구획S-parameter
ADVAN1 (IV 1구획)CentralOutput—CMT1S1 = V
ADVAN2 (경구 1구획)DepotCentralOutputCMT2S2 = V
ADVAN3 (IV 2구획)CentralPeripheralOutputCMT1S1 = V1
ADVAN4 (경구 2구획)DepotCentralPeripheralCMT2S2 = V2
💡투여 경로를 몸의 흐름으로: ADVAN3(IV): 주사→혈관(CMT1)→말초(CMT2). ADVAN4(경구): 입→위장(Depot=CMT1)→혈관(CMT2)→말초(CMT3). 경구 흡수라는 추가 단계가 Depot을 CMT1에 밀어넣는다. 이 한 칸 밀림이 S1→S2, V1→V2 등 모든 번호 오프셋의 원인이다.

B-2. S-parameter 단위 통일 공식

$$S_n = V \times \text{usv}, \quad \frac{\text{Amount}}{V \times \text{usv}} = \text{DV 단위}$$

dose(mg), V(L), Cp(ng/mL) → $\frac{\text{mg}}{L \cdot \text{usv}} = \frac{\text{ng}}{\text{mL}} \Rightarrow \text{usv} = \frac{1}{1000} \Rightarrow S2 = \frac{V2}{1000}$ [Owen Eq.3.6, p.41]

💡S-parameter의 본질: NONMEM 내부 농도 계산은 항상 Amount ÷ Sn이다. Sn 미정의 시 기본값 1 적용 → 단위 불일치가 IPRED에 그대로 반영 — 에러 메시지 없이.

ADVAN4 TRANS4 완전 코드

$SUBROUTINE ADVAN4 TRANS4

$PK
  TVCL = THETA(1)           ; 집단 청소율 (L/h)
  TVV2 = THETA(2)           ; 중심구획 분포용적 (L)
  TVV3 = THETA(3)           ; 말초구획 분포용적 (L)
  TVQ  = THETA(4)           ; 구획간 청소율 (L/h)
  TVKA = THETA(5)           ; 흡수속도상수 (1/h)

  CL = TVCL * EXP(ETA(1))  ; 로그정규 IIV — 양수 항상 보장
  V2 = TVV2 * EXP(ETA(2))
  V3 = TVV3 * EXP(ETA(3))
  Q  = TVQ  * EXP(ETA(4))
  KA = TVKA * EXP(ETA(5))

  ALAG1 = THETA(6) * EXP(ETA(6))  ; 흡수지연 (CMT1=Depot에 적용)

  S2 = V2/1000              ; Dose(mg)/V(L)→Cp(ng/mL) 단위 통일 ★핵심
  ; TRANS4 내부 자동: k10=CL/V2, k23=Q/V2, k32=Q/V3

  KE  = CL/V2               ; 출력용 별칭 (PREDPP 예약변수 K 재정의 아님)
  K23 = Q/V2
  K32 = Q/V3

[출처: PIPET, Ch.3 코드 3.2, p.32] [출처: Owen, Ch.3, pp.36-41]

ADVAN3 → ADVAN4 전환 필수 체크리스트

⚠️동시에 수정해야 할 항목은 정확히 4가지다. 하나라도 빠뜨리면 NONMEM은 에러 없이 잘못된 결과를 낸다: ① ADVAN 번호(3→4) ② S-parameter(S1→S2) ③ 변수명(V1/V2→V2/V3, KA 추가) ④ 데이터셋 CMT 값(관찰 기록 CMT=1→2). 특히 ④는 제어구문 바깥 파일에 있어 가장 놓치기 쉽다.
; BEFORE (ADVAN3 IV 2구획)       AFTER (ADVAN4 경구 2구획)
$SUBR ADVAN3 TRANS4            → $SUBR ADVAN4 TRANS4
  V1 = THETA(2)*EXP(ETA(2))   →   V2 = THETA(2)*EXP(ETA(2))
  V2 = THETA(3)*EXP(ETA(3))   →   V3 = THETA(3)*EXP(ETA(3))
                               →   KA = THETA(5)*EXP(ETA(5))  ; 추가
  S1 = V1/1000               →   S2 = V2/1000               ; ★핵심
; 데이터셋: 관찰 CMT=1         → 데이터셋: 관찰 CMT=2          ; ★핵심
C-2 Plausible Fallacy — Apex Concept ⚡

오류 형태: ADVAN4 전환 후 S2 = V2/1000 미기재 → NONMEM S2 기본값=1 적용.

수학적 나비효과:

$$\text{IPRED}_{\text{잘못}} = \frac{A(2)}{1}, \quad \text{IPRED}_{\text{올바른}} = \frac{A(2)}{V2/1000}$$

$$\text{오차 비율} = \frac{V2}{1000} \xrightarrow{V2=50\text{L}} \frac{50}{1000} = 0.05 \quad \Rightarrow \textbf{ 20배 과소추정}$$

NONMEM은 이 체계적 오류를 보상하기 위해 $SIGMA·$OMEGA를 비정상적으로 크게 추정하며 $COV step까지 성공한다.

GOF 시그니처 "S-parameter 망각 패턴": DV vs IPRED에서 IPRED가 DV 대비 V2/1000배 체계적 과소추정 (전 농도 범위 균일), $SIGMA/$OMEGA 비정상 팽창, BOUNDARY 경고 다수.

CARD 03 $ERROR 잔차 구조 심화 — 가법/비례/혼합/log-error/assay-specific
개념 Big Idea: 거장의 시각

왜 치명적으로 중요한가: 3상 풀링 데이터에서 Phase 1과 Phase 3을 하나의 잔차 구조로 모델링하면 FDA 심사관이 "분석법 차이를 무시한 모델"로 지적하고 $OMEGA가 과대추정되어 시뮬레이션 전체가 오염된다.

체화해야 할 직관: CWRES vs IPRED 플롯이 $ERROR 구조 선택의 지문이다 — 깔때기(funnel)이면 가법 오명세, 역깔때기이면 CCV 오명세.

B. 주요 잔차 구조

; 1. 가법 — 저농도, LLOQ 근처
Y = F + EPS(1)                     ; Var(Y) = σ₁²

; 2. 비례(CCV) — 고농도, 상대 오차 일정
Y = F * (1 + EPS(1))               ; %CV = σ₁×100

; 3. 혼합 — 전 농도 범위
Y = F + EPS(1) + F*EPS(2)         ; Var(Y) = σ₁² + F²·σ₂²

; 4. W함수 + $SIGMA 1 FIX — 현장 표준
$ERROR
  IPRED = F
  W     = SQRT(THETA(3)**2 + THETA(4)**2 * IPRED**2)
  IRES  = DV - IPRED
  IWRES = IRES / W
  Y     = IPRED + W * EPS(1)
  ; THETA(3)=가법 SD(ng/mL), THETA(4)=비례 CV(무단위)
  ; → 보고서에 추가 변환 없이 바로 사용 가능 ← 현장 표준 이유

[출처: PIPET, Ch.3, pp.35-37] [출처: Owen, Ch.3, pp.44-49]

💡W함수 + $SIGMA 1 FIX가 현장 표준인 이유: $SIGMA로 분산을 추정하면 보고 시 반드시 √를 취해야 한다. W함수 패턴에서는 THETA(3)=가법 SD, THETA(4)=비례 CV — 추가 변환 없이 보고서에 바로 사용 가능.

%CV 도출 및 핵심 오류 경고

$Y = F(1+\varepsilon)$에서: $\text{Var}(Y) = F^2\sigma_1^2 \Rightarrow SD(Y) = F\sigma_1 \Rightarrow \%CV = \sigma_1 \times 100$

⚠️$SIGMA 0.04 = %CV 4%? 아니다, 20%다! $SIGMA에 입력하는 숫자는 분산(variance)이다. $\sqrt{0.04}=0.2 \rightarrow \%CV=20\%$. 이 오류로 실제의 5배 과소보고 시 FDA 심사관의 수학적 재검토에서 즉시 발각된다. 역산 기억법: %CV 30%를 원하면 $SIGMA = $(0.3)^2 = 0.09$. [Owen, p.53]

Log-error 방어 코딩 3가지 패턴

; 패턴 1: OBSERVATIONS ONLY (Owen 표준)
$ERROR (OBSERVATIONS ONLY)
  Y = LOG(F) + EPS(1)
  ; 투여 기록(EVID=1)에서 F≈0 → LOG(0) 수학 오류 방지

; 패턴 2: CALLFL=0 (동일 효과)
  CALLFL = 0
  Y = LOG(F) + EPS(1)

; 패턴 3: FLAG (가장 안전 — 극저농도 F≈0에도 대응)
  FLAG = 0
  IF (F.EQ.0) FLAG = 1
  Y = (1-FLAG)*LOG(F+FLAG) + EPS(1)
  ; ⚠ FORTRAN LOG() = 자연로그(ln), 상용로그 = LOG10()

[출처: Owen, Ch.3, pp.46-47]

💡왜 투여 기록에서 F=0이 되는가? PREDPP는 투약 기록(EVID=1)에서 구획에 약물을 추가하는 이벤트로 처리하고, 투약 직후 흡수 전이므로 F≈0이 나올 수 있다. LOG(0)은 -∞ → NONMEM 즉시 중단. OBSERVATIONS ONLY는 "관찰 기록에서만 $ERROR 실행, 투여 기록은 건너뛰어라"는 지시다.

B. $ESTIMATION 핵심 옵션

$ESTIMATION NOABORT MAXEVAL=9999 METHOD=1 INTER PRINT=5 SIGDIGITS=3

; METHOD=0: FO — η=0에서 테일러 전개, 빠르나 편향 (규제 제출 비권장)
; METHOD=1: FOCE — 개인별 사후 ETA에서 전개, 현장 표준
; METHOD=1 INTER (FOCEI): ETA-EPS 상호작용 포함, 비례/혼합 잔차 시 필수

CARD 05 $PRED — PREDPP 없는 자유 코딩과 PD/QT 모델링

왜 치명적으로 중요한가: ICH E14 QTc-노출 분석, Emax PD 모델, 비선형 바이오마커 반응은 ADVAN으로 표현 불가능한 비PK 관계를 NONMEM으로 추정해야 하는 규제 표준 분석이다.

체화해야 할 직관: $PRED는 NONMEM에게 "PK 라이브러리를 내려놓고 내가 직접 예측식을 쓸게"라는 선언이다 — 대신 AMT/CMT 같은 PREDPP 약속어가 사라지고, 투여량을 매 관찰 행마다 공변량처럼 반복 기재해야 한다.

B. QTc 선형 모델 — $PRED 최소 예시

$PROB QTcF-Cmax 분석 — ICH E14
$INPUT ID=DROP DQTCF=DV CMAX
$DATA qtc.csv IGNORE=C
$PRED
  INT = THETA(1)             ; Intercept (ms)
  SLP = THETA(2)             ; Slope (ms per ng/mL)
  EFF = SLP*CMAX + INT
  Y   = EFF + ETA(1)         ; $ERROR 없이 $PRED 내부에서 Y 직접 정의
$THETA 2.0  0.002
$OMEGA 4.0
$SIGMA 9.0                   ; 보고: SD = √9 = 3ms (NOT 9ms!)
$EST METHOD=1 INTER MAXEVAL=9999 PRINT=5

[출처: PIPET, Ch.3, 코드 3.8, p.46]

💡"D를 모든 관찰 행에 반복 기재"가 필요한 이유: PREDPP가 없으면 "투약 후 시간 흐름에 따라 적분 유지"하는 엔진이 없다. 해석해 수식이 "t시간 후 남은 양"을 직접 계산하기 때문에, 각 관찰 시점에서 초기 투여량(D)만 알면 된다.

Sigmoidal Emax PD 모델

$PRED
  E0   = THETA(1)
  EMAX = THETA(2)
  C50  = THETA(3)*EXP(ETA(1))   ; EC50 — IIV 포함
  GAM  = THETA(4)                ; Hill 계수 — ETA 추가 금지!

  IPRD = E0 + (EMAX-E0)*CP**GAM / (CP**GAM + C50**GAM)
  Y    = IPRD + EPS(1)

[출처: PIPET, Ch.7, 코드 7.1, p.71]

📌Hill 계수(GAM)에 ETA 추가하지 않는 이유: GAM이 지수에 들어가기 때문에(CP^GAM), 개인별 변이가 있으면 OFV gradient가 극도로 불규칙해져 수렴이 불안정해진다. 현장 표준은 GAM을 집단 고정값으로만 추정.
aliases:[$PRED, PREDPP 없는 코딩, Emax PD, QTc 노출, Hill 계수]
$PRED는 PREDPP를 대체하여 PD·비PK·해석해 PK를 자유 수식으로 구현한다. 투여량을 모든 관찰 행에 반복 기재, $ERROR 없이 $PRED 내부에서 Y 직접 정의. GAM에 IIV 추가는 현장 표준에서 금지.

구조적 동형성 지도

통찰의 서막 — 구조적 동형성의 Big Idea

공통 원리: NONMEM 제어구문의 순서 의존성은 "전방 선언(forward declaration) 필요성"이라는 보편 원리에서 비롯된다. 뒤에 오는 레이어는 앞 레이어의 출력물을 소비해야만 동작하며, 이 단방향 의존성(DAG: directed acyclic graph)이 컴파일러, SQL, OSI 네트워크 스택, NONMEM 파서 모두에서 동일한 "층화 순차 처리(layered sequential processing)" 패턴을 만든다.

왜 이 형태인가: 각 레이어가 이전 레이어의 출력을 전제로 자신의 변환을 수행하기 때문에, 순서를 바꾸면 참조 대상이 미정의 상태가 되어 전체 시스템이 무결하지 않다. 이것은 수학적으로 위상 정렬(topological sort)의 필연성이다.

예측 전이: SQL에서 WHERE절이 FROM절 이전에 실행될 수 없는 이유를 이해하면, NONMEM $PK가 $INPUT 이전에 올 수 없는 이유를 별도로 외울 필요가 없다 — 같은 구조다.

진단 병리 — 2개 GOF 시그니처

🔴 시그니처 1: "S-parameter 망각 패턴 (Sn Amnesia)"

GOF 패턴: DV vs IPRED에서 IPRED가 DV 대비 V/1000배 체계적 과소추정. 전 농도 범위에서 균일한 수직 편향. 1:1 직선 대비 모든 점이 아래쪽 집중.

기계론적 원인: S-parameter 미정의 → 기본값 S=1 → IPRED = A(central)/1 → V/1000배 과소추정. NONMEM은 이를 보상하기 위해 $SIGMA·ETA를 비정상적으로 크게 추정.

해결: $PK에 S2 = V2/1000 추가. ADVAN3→ADVAN4 전환 체크리스트 4항목 재점검. FDATA로 CMT 분포 검증.

η-shrinkage: 체계적 IPRED 편향 존재 시 ETA 분포가 정규에서 이탈, η-shrinkage > 30% 비정상.

🔴 시그니처 2: "$SIGMA %CV 오인 패턴 (Variance Confusion)"

GOF 패턴: CWRES vs IPRED 무작위 — 모델 구조 자체는 올바름. 그러나 보고서의 RV(%CV) 값이 실제의 1/√σ² 수준으로 과소 기재.

기계론적 원인: $SIGMA에 입력된 값이 분산임을 모르고 %CV로 직접 보고. 예: $SIGMA 0.09 → 실제 %CV = √0.09×100 = 30% → 잘못 보고: "9%".

해결: 반드시 $\%CV = \sqrt{\sigma^2} \times 100$ 변환 적용. W함수 + $SIGMA 1 FIX 패턴으로 전환하면 THETA(4)가 직접 CV값으로 추정되어 변환 불필요. ε-shrinkage > 30% 시 잔차 구조 재검토.

C. Professional Moat

NONMEM을 실행하고 파라미터를 추출하는 것은 이미 자동화 가능하다. 이 세션의 제어구문 구조와 진단 논리를 진정으로 내면화한 모델러만이 할 수 있는 것은 세 가지다.

첫째, 침묵의 오류를 코드 한 줄로 진단한다. 모든 IPRED가 DV보다 수십 배 낮은 상황 앞에서 "S-parameter 단위 불일치"를 즉시 식별하고 수정한다. 알고리즘은 오류 메시지가 없으면 이 상황을 "정상 수렴"으로 분류한다.

둘째, 교과서의 오기를 현장에서 검증한다. PIPET Table 4.5 ADVAN3 CMT 오기를 Owen과 대조하여 식별하고, IGNORE AND/OR 불일치를 엔진 동작 기준으로 해소한다. 이 비판적 교차검증 능력은 단일 소스에 의존하는 알고리즘과 근본적으로 다르다.

셋째, Condition number와 상관행렬을 읽어 모델 선택을 규제 언어로 방어한다. ρ(V3,Q)=0.96이 "말초 분포 데이터가 없다"는 생리학적 독해이고, 이것이 "2구획 축소" 또는 "Q를 FIX"라는 처방으로, 그리고 "Phase 2 재평가"라는 규제 언어로 이어지는 판단의 사슬을 알고리즘은 수행하지 못한다.
