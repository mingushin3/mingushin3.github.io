Q01 Protocol 숙지 & Target Model 구조 역설계
MUSTv3
데이터를 만지기 전에 먼저 확인: (1) 이 연구의 투여 설계(single/multiple dose, dose levels, 투여 경로, 투여 간격)를 알려줘. (2) 구축할 Base Model 구조(몇 compartment, 흡수 모델, estimation method)를 결정하거나 계획해줘. (3) 이에 따른 CMT 번호 체계, EVID 사용 패턴, 필수 공변량 목록을 사전 정의해줘. Protocol/SAP 문서가 있으면 첨부.
🎯 LLM 출력물
① 연구 설계 요약 (투여 설계, dose levels, 채혈 스케줄, PK/PD endpoints)
② Target Model 구조 선언 (compartments, absorption, 예상 $PK 구조)
③ CMT 번호 사전 할당 테이블 (dose CMT, obs CMT, PD CMT 등)
④ 필수 공변량 사전 목록 (protocol/SAP 기반)
⑤ EVID 사용 패턴 예상 (EVID=0,1만? 2,3,4도 필요?)
🔨 장인 노트
가장 뛰어난 전처리 전문가는 R 스크립트를 열기 전에 NONMEM Control Stream을 먼저 머릿속에 그린다. 데이터에서 출발하는 것이 아니라, 최종 모델이 요구하는 구조에서 출발해 데이터를 맞춘다. 이것이 "역설계(Reverse Engineering)"이며, 이 단계를 건너뛰면 나중에 CMT를 재할당하고, 컬럼을 추가하고, 데이터를 재구성하는 재작업이 발생한다.
→ 장인 문서: 사고 과정 (역설계) / Map Phase 1: CMT 구조 결정의 근거
Q02 데이터 형태 식별 & 파일 인벤토리
MUST
업로드한 데이터 파일들을 확인하고, (1) 파일 형식 (xpt/csv/xlsx/sas7bdat), (2) 데이터 유형 (SDTM/ADaM/Raw CRF/Excel/기존 NONMEM), (3) 각 파일의 행·열 수와 변수 목록을 표로 정리해줘. SDTM이면 도메인명과 SDTM version(3.2/3.3/3.4)도 식별. ADaM이면 SDTM도 함께 요청해야 하는지 판단.
🎯 LLM 출력물
① 파일 인벤토리 테이블 (파일명 / 형식 / 유형 / 행수 / 열수 / 핵심 변수)
② 데이터 유형 판정 + SDTM version (해당시)
③ ADaM일 때 SDTM 추가 요청 필요성 판단
④ 후속 COND 질문 중 건너뛸 수 있는 질문 사전 식별
🔨 장인 노트
SDTM version에 따라 변수명이 미묘하게 다를 수 있다 (예: 3.2 vs 3.4). ADaM은 통계 분석용이므로 NONMEM 구조와 다르다 — ADaM이 있어도 SDTM을 함께 확보하는 것이 안전하다. ADSL의 baseline covariates는 직접 활용 가치가 높다.
→ Map Phase 0: 전체 소스 유형 분기 / SDTM·ADaM·Raw CRF·Excel
Q03 Subject ID 식별 & 고유 대상자 수 확인
MUST
데이터에서 대상자 식별 변수(USUBJID, SUBJID, PTNO 등)를 찾고, 고유 대상자 수를 세어줘. ID 형식이 문자열이면 숫자 ID로 재코딩하는 매핑 테이블도 만들어줘. 여러 파일에서 ID 교집합/차집합도 확인. "001" vs "1" vs "S-001" 같은 형식 불일치도 점검.
🎯 LLM 출력물
① ID 매핑 테이블 (원본ID → 숫자ID)
② 파일 간 ID Venn diagram 정보 (교집합/차집합 수)
③ R 코드: id_map <- dm %>% distinct(USUBJID) %>% mutate(ID = row_number())
④ ID 형식 불일치 경고 (있다면)
→ Map Phase 0: Subject ID 체계 통일
Q04 날짜/시간 변수 식별 & 형식 평가
MUST
모든 파일에서 날짜/시간 관련 변수를 찾고, 각각의 형식(ISO 8601? AM/PM? 날짜만?), 결측률, 파싱 가능 여부를 표로 정리해줘. PK 분석이면 반드시 datetime(분 단위 이상) 필요. PD-only면 date만으로 충분할 수 있음.
🎯 LLM 출력물
① 날짜/시간 변수 인벤토리 테이블 (변수명 / 형식 / 정밀도 / 결측률 / 파싱가능여부)
② PK vs PD 시간 정밀도 충분성 판정
③ AM/PM, locale 혼재 시 변환 전략 제안
④ 날짜+시간 분리 여부 판정
🔨 장인 노트
PK 분석에서 시간 정밀도 = 분석 정밀도. 채혈-투여 간격이 분 단위로 PK 파라미터에 영향한다. 시간이 없는 PK 데이터는 imputation이 불가피하며, imputed record는 반드시 FLAG로 표시해야 한다.
→ Map Phase 2: Date vs Datetime 결정 / Phase 2 선행 조건
Q05 핵심 소스 테이블 매핑
MUST
NONMEM building block 3개의 소스를 매핑해줘:
• Dosing 소스: 어떤 파일/테이블의 어떤 변수가 투여 기록인가?

① 3-block 매핑 테이블 (소스 → NONMEM 변수)
② DDT 초안 (Raw CRF/Excel인 경우) — "어떤 소스 변수 → 어떤 NONMEM 변수, 어떤 변환 규칙"
③ 누락된 핵심 변수 경고 (있다면)
→ Map Phase 0: DDT 수립 / 6대 표준 입력 확인
Q06 기존 NONMEM 데이터셋 인수인계 체크
COND기존NM만
이 데이터셋은 기존 NONMEM 데이터셋이다. (1) TIME 기준점을 역추적해줘. (2) FLAG/IGNORE/EXCL 컬럼 convention을 파악해줘. (3) $INPUT 변수 순서를 데이터 컬럼 순서와 대조해줘. (4) 새 데이터 추가 시 ID 충돌 방지 방안도 제안해줘.
🎯 LLM 출력물

③ FLAG/EXCL convention 요약
④ ID 충돌 방지 전략 (offset 등)
⏭ 기존 NONMEM 데이터를 수정/확장하는 경우에만 해당. 새로 만드는 경우 건너뛰기.

Q07 Multi-study pooling 전략 수립
COND다수study

⑤ Study별 대상자수/투여횟수 요약

Q08 투여 기록 추출 & 기본 구조 생성
MUST
dosing 소스에서 투여 기록을 추출해서 다음 컬럼을 가진 dosing 데이터프레임을 만들어줘: ID, DATETIME, AMT, CMT, EVID=1, MDV=1, DV=0. 제외 기준 적용: EXSTAT="NOT DONE", screening failure, documented protocol deviation으로 투여 무효 처리된 건도 제외. 각 제외 사유별 record 수 보고.
🎯 LLM 출력물
① dosing 데이터프레임 구조 확인 (head(), str())
② ID별 투여 횟수 요약 (min/median/max)
③ 제외 사유별 record 수 보고 (NOT DONE / screening failure / protocol deviation)
④ R 코드 전문
🔨 장인 노트
장인은 단순히 "NOT DONE"만 제외하지 않는다. 임상시험의 물리적 현실을 이해한다: 부분 주입이 중단된 경우, 투여 후 구토한 경우, protocol 위반으로 투여가 무효화된 경우 — 각각 다른 처리가 필요하며, 모든 제외 사유를 R 코드로 명시적으로 기록한다.
→ Map Phase 1: Dosing records 필수 변수
Q09 Actual vs Planned dose 확인
MUST
AMT에 들어간 값이 실제 투여량(actual dose)인지 계획 투여량(planned dose)인지 확인해줘. Actual과 planned가 다른 record를 찾아 목록으로 보여줘. AMT는 반드시 actual dose여야 하고, planned dose는 별도 PDOSE 컬럼으로 보존.
🎯 LLM 출력물
① Actual ≠ Planned 불일치 record 목록 (ID, TIME, actual, planned)
② 불일치 비율 (%) 및 원인 추정 (부분주입, 구토, dose reduction 등)
③ PDOSE 컬럼 생성 R 코드
→ Map Phase 1: Actual vs Planned dose
Q10 투여 경로 & RATE/DUR 설정
MUST
투여 경로(PO/IV bolus/IV infusion/SC/IM)를 확인하고, 경로에 맞게 RATE와 DUR을 설정해줘:
• 경구/SC/IM bolus → RATE=0
• IV bolus → RATE=0
• IV infusion → RATE=-2 + DUR=주입시간(hr)
혼합 경로이면 각각 다른 CMT 할당 고려. DUR 계산 시 EXSTDTC~EXENDTC 차이 사용.
🎯 LLM 출력물
① 투여경로 분포 테이블 (경로 / record수 / %)
② RATE, DUR 컬럼 설정 R 코드
③ IV infusion의 DUR 계산 로직
→ Map Phase 1: 투여 경로별 의사결정 트리
Q11 AMT 단위 확정 & 변환
MUST
AMT의 단위를 확인하고 확정(mg? μg? cells?). DV 단위와의 조합이 PK 파라미터 단위를 결정: AMT(단위) ÷ DV(단위) → CL(단위). 단위 변환이 필요하면 수행하고 변환 로직을 기록.
🎯 LLM 출력물
① 단위 관계식: AMT(___) / DV(___) → CL(___), V(___)
② 단위 변환 R 코드 (필요시)
③ AMT 범위 요약 (min, median, max, unique values)
→ Map Phase 1: 단위
Q12 체중 기반 투여 → absolute dose 변환
CONDmg/kg투여
투여가 mg/kg 기반이면: AMT를 absolute dose(mg)로 변환. 예: 10mg/kg × 70kg = AMT=700. 계산에 사용된 체중은 WT 컬럼으로 별도 보존. WT 결측인 ID 경고.
🎯 LLM 출력물
① 변환 R 코드: mutate(AMT = AMT_MGKG * WT)
② 변환 전후 AMT 비교 요약
③ WT 결측인 ID 경고
⏭ Flat dose (mg 기반) 투여이면 건너뛰기.
→ Map Phase 3: 체중 기반 투여

Q14 반복 투여 압축 (ADDL/II)
COND반복투여
동일 용량의 반복 투여 구간을 찾아서 ADDL과 II(hr)로 압축해줘. 용량 변경 시점에서는 ADDL을 끊고 새 dosing record를 시작. 압축 전후 총 투여 횟수가 동일한지 검증. Unit test: 가상 환자 1명을 만들어 코드가 정확히 의도대로 ADDL/II를 생성하는지 먼저 테스트한 후 전체 적용.
🎯 LLM 출력물
① 압축 전 총 투여 record 수 vs 압축 후 record 수
② 총 투여 횟수 동일성 검증 결과
③ 용량 변경 시점 식별 목록
④ 가상 환자 unit test 결과
