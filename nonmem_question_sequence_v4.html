Q01 내가 짜야 할 NONMEM Control Stream은 어떤 구조인가?
MUST
🔍 이 질문의 의의 — 왜 이걸 가장 먼저 해야 하는가
장인은 R 스크립트를 열기 전에 NONMEM Control Stream을 머릿속에 먼저 그린다. 왜냐하면 CMT 번호 체계, EVID 사용 패턴, 필수 공변량 목록이 모두 내가 구축할 모델 구조에 의해 결정되기 때문이다.

• 경구 1-comp 모델이면 → CMT=1(depot), CMT=2(central), 흡수 파라미터 KA 필요
• IV 2-comp 모델이면 → CMT=1(central), RATE/-2 세팅, 분포 파라미터 Q/V2 필요
• PK+PD 통합이면 → PD endpoint용 별도 CMT 또는 DVID 할당 필요

이 단계를 건너뛰면 나중에 CMT를 재할당하고, 컬럼을 추가하고, 데이터를 재구성하는 재작업이 반드시 발생한다.
📋 R 코드 요청 프롬프트 (LLM에 복사해서 붙여넣기)
이 연구의 protocol/SAP 정보를 기반으로 다음을 정리해줘:
(1) 투여 설계 요약: single/multiple dose, dose levels, 투여 경로, 투여 간격, 채혈 스케줄
(2) 구축할 Base Model 구조 제안: compartment 수, 흡수 모델, 예상 $SUBROUTINES/$PK 구조
(3) CMT 번호 사전 할당 테이블: dose CMT, obs CMT, PD CMT(해당시)
(4) EVID 사용 패턴 예상: EVID=0,1만 쓸 건지, 2/3/4도 필요한지
(5) 필수 공변량 사전 목록: protocol/SAP에서 지정된 것 + 일반적으로 필요한 것
R 코드가 아니라 분석 계획 요약 테이블로 정리해줘.
✅ 완료 조건
□ CMT 번호 매핑 테이블이 확정되었다 (dose CMT, obs CMT)
□ EVID 사용 패턴이 결정되었다 (0/1만? 2/3/4도 필요?)
□ 필수 공변량 목록이 나열되었다
□ 예상 모델 구조가 선언되었다 (1-comp? 2-comp? 흡수 모델?)
🎯 LLM이 산출해야 할 결과물
① 연구 설계 요약표
② Target Model 구조 선언
③ CMT 번호 사전 할당 테이블
④ EVID 사용 패턴 예상
⑤ 필수 공변량 사전 목록
🔨 장인 노트 — 의사결정 트리
경구 투여인가?
→ Yes: depot compartment(CMT=1) + central compartment(CMT=2). $SUBROUTINES ADVAN2 TRANS2 (1-comp) 또는 ADVAN4 TRANS4 (2-comp). KA가 $PK에 필요.
→ No, IV bolus: central만. ADVAN1 TRANS2. RATE=0.
→ No, IV infusion: central만. ADVAN1 TRANS2. RATE=-2 + DUR 컬럼 필요.

PD endpoint도 함께 분석하는가?
→ Yes: PD endpoint용 별도 CMT 할당 또는 DVID 컬럼으로 구분. $ERROR에서 DVID 분기.
→ No: CMT는 dosing/obs 2개면 충분.

이것이 전처리의 첫 단추다. 여기서 결정된 CMT 체계가 Q13(dose CMT), Q23(obs CMT)에서 그대로 적용된다.

Q03 대상자는 몇 명이고, ID를 숫자로 변환할 수 있는가?
MUST
🔍 의의
NONMEM의 ID는 반드시 양의 정수여야 한다. 원본의 "S-001", "ABC-0042" 같은 문자열 ID를 숫자로 재코딩해야 하며, 여러 파일에서 ID가 일치하는지(교집합/차집합) 확인해야 한다. 여기서 ID 매핑이 틀리면 전체 데이터가 뒤섞인다.
📋 R 코드 요청 프롬프트
다음 R 코드를 짜줘:
(1) 모든 파일에서 대상자 식별 변수(USUBJID, SUBJID, PTNO 등)를 찾아서 distinct()로 고유 대상자 수를 세줘
(2) ID가 문자열이면 숫자 ID로 매핑: id_map <- dm %>% distinct(USUBJID) %>% mutate(ID = row_number())
(3) 여러 파일의 ID 교집합/차집합: anti_join()으로 파일 A에만 있는 ID, 파일 B에만 있는 ID 목록
(4) ID 매핑 테이블을 id_map으로 저장
✅ 완료 조건
□ 총 고유 대상자 수가 확정되었다
□ 원본ID → 숫자ID 매핑 테이블(id_map)이 생성되었다
□ 파일 간 ID 불일치가 0건이거나, 불일치 원인이 파악되었다

Q04 날짜/시간은 어디에, 어떤 형식으로 기록되어 있는가?
MUST
🔍 의의
PK 분석에서 시간 정밀도 = 분석 정밀도다. 날짜만 있고 시간이 없으면 TIME 계산 자체가 불가능하다. AM/PM 형식, locale 혼재, 날짜+시간 분리 등을 이 단계에서 미리 파악해야 Phase 2(TIME 계산)에서 사고를 방지한다.
📋 R 코드 요청 프롬프트
모든 파일에서 날짜/시간 관련 변수를 찾아서 다음 테이블을 R로 만들어줘:

(2) 파싱 테스트: 대표 값 5개를 lubridate::ymd_hms() 등으로 파싱 시도, 성공/실패 보고
(3) 날짜와 시간이 별도 컬럼이면 합치는 코드 제안
✅ 완료 조건
□ 모든 날짜/시간 변수의 형식이 파악되었다
□ 파싱 전략이 결정되었다 (어떤 lubridate 함수를 쓸 것인가)
□ 시간이 결측인 변수가 있다면 목록이 정리되었다

Q05 Dosing·Observation·Covariate의 소스는 각각 어느 파일/변수인가?
MUST
🔍 의의
NONMEM 데이터셋은 3개의 building block으로 조립된다: Dosing(EVID=1), Observation(EVID=0), Covariates. 각각의 소스가 어느 파일의 어느 변수인지 매핑해야 Q08~Q50까지의 전체 경로가 결정된다. Raw CRF/Excel이면 DDT(Data Definition Table)를 여기서 작성해야 한다.

Q06 기존 NONMEM 데이터셋을 이어 쓰는 것인가?
COND
🔍 의의
이전 분석자의 데이터셋을 인수인계 받거나 새 코호트를 추가할 때는, 기존 데이터의 TIME 기준점, FLAG convention, $INPUT 순서를 먼저 파악해야 한다. 기준점이 다르면 모든 TIME이 어긋난다.
📋 R 코드 요청 프롬프트
기존 NONMEM 데이터셋(기존파일명.csv)을 R로 읽고 다음을 분석해줘:
(1) TIME 기준점 역추적: 각 ID의 첫 TIME=0인 record가 첫 dosing인지 확인
(2) FLAG/IGNORE/EXCL 컬럼 convention: names(df) %>% str_subset("FLAG|EXCL|IGNORE|SKIP")
(3) $INPUT 변수 순서 vs 데이터 컬럼 순서 대조: paste(names(df), collapse=" ")로 $INPUT 문자열 재구성
(4) 기존 max ID + offset으로 새 ID 충돌 방지 코드
✅ 완료 조건
□ TIME reference가 역추적되었다
□ $INPUT 순서가 파악되었다
□ ID offset 전략이 결정되었다
⏭ 건너뛰기

Q07 여러 Study를 합쳐야 하는가?
COND
🔍 의의
Multi-study pooling은 AMT/DV 단위 불일치, assay(분석법) 차이에 의한 LLOQ 차이, ID 충돌이라는 3대 함정이 있다. STUDYID 컬럼은 모델에서 study effect를 넣을 수 있게 해주는 핵심 변수다.
📋 R 코드 요청 프롬프트
2개 이상 study를 pooling하는 R 코드를 짜줘:
(1) STUDYID 컬럼 생성: mutate(STUDYID = case_when(...))
(2) AMT/DV 단위 일치 확인: study별 AMT/DV 범위 비교 테이블
(3) LLOQ 차이: study별 LLOQ 값 비교
(4) ID offset: Study B의 ID = Study A max ID + 1000
(5) Study별 요약: 대상자수, 투여횟수, obs수
✅ 완료 조건
□ STUDYID가 숫자로 코딩되었다
□ 단위 일치가 확인/변환되었다
□ ID가 study 간 충돌하지 않는다
⏭ 건너뛰기
단일 study → Q08로 바로 이동. STUDYID=1로 고정.

Q08실제로 약이 투여된 기록만 골라낼 수 있는가?
MUST
🔍 의의
EX 도메인이나 투여 기록에는 EXSTAT="NOT DONE"(투여 안 됨), screening failure, protocol deviation 등 실제로 투여되지 않은 기록이 섞여 있다. 이것들이 들어가면 NONMEM이 없는 투여를 있다고 계산한다. 장인은 제외할 때 모든 제외 사유를 R 코드로 명시적으로 기록한다.
📋 R 코드 요청 프롬프트
dosing 소스([파일/테이블명])에서 투여 기록을 추출하는 R 코드를 짜줘:
(1) 다음 컬럼을 가진 dose_df 데이터프레임 생성: ID, DATETIME, AMT, CMT, EVID=1, MDV=1, DV=0
(2) 제외 기준 적용: EXSTAT=="NOT DONE", screening failure, protocol deviation으로 투여 무효 건 제외
(3) 제외 사유별 record 수 보고: count(제외사유)
(4) ID별 투여 횟수 요약: dose_df %>% count(ID) %>% summary()
✅ 완료 조건
□ dose_df가 생성되었다
□ 제외된 record 수와 사유가 보고되었다
□ 모든 ID에 최소 1개 투여 기록이 있다
🔨 장인 노트
장인은 단순히 "NOT DONE"만 필터링하지 않는다. 부분 주입이 중단된 경우(예: IV infusion 중 알레르기 반응) → actual dose가 planned dose보다 적음. 투여 후 구토 → dose가 불완전하게 흡수됨. 이런 경우 제외가 아니라 AMT를 actual dose로 조정해야 한다. 이것은 Q09에서 처리한다.
Q09AMT가 실제 투여량(actual)인가, 계획 투여량(planned)인가?
MUST
🔍 의의
AMT에 planned dose가 들어가면 CL/V 추정이 체계적으로 편향된다. 100mg을 투여할 예정이었지만 실제 80mg만 투여된 환자의 AMT가 100이면, NONMEM은 "100mg에 비해 농도가 너무 낮다" → CL을 과대추정한다. Top 3 실무 실수 중 하나다.
