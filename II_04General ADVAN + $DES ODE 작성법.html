개념 해부 카드

일반 ADVAN 분류 체계 — 선택의 논리
개념 Big Idea: 거장의 시각

왜 치명적으로 중요한가: 잘못된 ADVAN을 선택하면 추정이 수렴하지 않거나, 수렴하더라도 내부적으로 수치적으로 불안정한 해를 반환한다 — ADVAN6로 stiff 단일클론항체를 추정하다 며칠을 날리는 것은 이 체계를 모르는 모델러의 전형적 경험이다.

체화해야 할 단 하나의 직관: ADVAN 선택은 "어떤 모델을 쓰느냐"가 아니라 "미분방정식 풀이기(solver)를 어느 것으로 쓰느냐"의 문제다. 같은 2구획 경구 모델을 ADVAN5로도, ADVAN6로도 쓸 수 있다. 차이는 수학적 해법의 종류이지 모델 구조 자체가 아니다.

이 직관을 머릿속에 박고 아래를 읽어라: 모델 선택(구획 수, 비선형성)과 ADVAN 선택(수치 풀이 전략)은 독립적인 두 개의 의사결정이다.

A. 분류 체계 (PIPET 표5.1, 5.2)

C. Structural Necessity

선형 ODE는 행렬 지수 $e^{\mathbf{K}t}$의 수치적 근사로 풀 수 있으므로 $PK의 K_mn 마이크로상수만으로 충분하다. 반면 비선형 ODE는 해석적 해가 존재하지 않으므로 수치 적분기(Runge-Kutta, Gear)가 요구되며, 이를 구동하기 위해 $DES 블록이 구조적으로 필수다.

실용 선택 기준 — ADVAN6 vs ADVAN8 (Owen 제안)

$ESTIMATION에 MAXEVAL=1을 설정한 후 ADVAN6와 ADVAN8을 각각 1회 반복 실행하여, 더 빠르게 성공하는 ADVAN을 선택한다.

선형 전이만 포함하면 ADVAN5/7, 비선형 전이가 포함되면 ADVAN6/8/9/13을 선택한다.
선택의 실질적 기준은 모델의 stiffness이며, 같은 모델을 다수의 ADVAN으로 구현 가능하므로
ADVAN 선택과 모델 구조 선택을 반드시 분리하여 사고해야 한다.
$MODEL 구획 선언 — COMP, DEFDOSE, DEFOBS
개념 Big Idea: 거장의 시각

왜 치명적으로 중요한가: DEFDOSE와 DEFOBS를 잘못 설정하면 NONMEM이 투여 기록과 관찰값을 엉뚱한 구획에 배정하며, 이 오류는 경고 없이 추정을 진행하고 OFV가 수렴하는 척 보이기 때문에 — 잘못된 파라미터가 규제 제출 문서에 올라가는 사고로 이어질 수 있다.

체화해야 할 단 하나의 직관: $MODEL의 COMP 선언은 NONMEM에게 "이 구획이 무엇을 받고, 무엇을 관찰하는 기본값인지" 알려주는 주소지 등록이다 — 데이터셋에 CMT 컬럼이 있으면 이 주소지는 덮어쓰이므로, 데이터셋의 CMT가 항상 최종 결정권을 가진다.

이 직관을 머릿속에 박고 아래를 읽어라: $MODEL은 CMT 컬럼이 없을 때의 보험이고, CMT 컬럼이 있으면 그것이 진짜 주소다.

B. Code Structure — $MODEL 완전 해부

$MODEL
    COMP(DEPOT,    DEFDOSE)    ; 1번 구획: 투여 기본 구획 (first-order 흡수)
    COMP(CENTRAL,  DEFOBS)     ; 2번 구획: 관찰값 기본 구획 (혈중 농도 측정)
    COMP(PERIPH)               ; 3번 구획: 말초 구획 (임의 명칭, 최대 999개)
[출처: PIPET Ch.5, pp.59–60; Owen Ch.9, §9.2, p.235]

DEFDOSE / DEFOBS 우선순위 규칙

상황NONMEM 동작
CMT 컬럼 없음 + DEFDOSE 지정DEFDOSE 구획에 투여
CMT 컬럼 없음 + DEFDOSE 없음 + 'DEPOT' 명칭첫 번째 DEPOT 구획에 투여
CMT 컬럼 없음 + 위 조건 없음1번 구획에 투여

C. Structural Necessity

$MODEL이 일반 ADVAN에서 필수인 이유: 특수 ADVAN은 구획 구조가 코드에 하드코딩되어 NONMEM이 K10, K12, K21을 자동으로 인식한다. 일반 ADVAN은 구획 수 자체를 모르므로, $MODEL이 없으면 $PK의 K12 정의가 무엇을 가리키는지 NONMEM이 해석 불가능하다.

$DES ODE 작성법
⚡ Apex Concept
개념 Big Idea: 거장의 시각

왜 치명적으로 중요한가: $DES는 NONMEM의 심장이다 — 여기서 ODE의 부호 하나가 틀리면(유입이 유출로, 또는 반대로) 추정은 수렴하되 생물학적으로 불가능한 파라미터를 반환하며, 이 오류는 GOF 플롯만으로는 잡히지 않는다.

체화해야 할 단 하나의 직관: $DES의 각 줄은 "이 구획에서 질량이 들어오는 것은 더하고(+), 나가는 것은 빼라(−)"는 질량 보존 원칙의 직역이다 — 시스템의 모든 구획에 대해 이 원칙만 충실히 따르면, 어떤 복잡한 ODE도 쓸 수 있다.

이 직관을 머릿속에 박고 아래를 읽어라: DADT(i) = (i로 들어오는 모든 항의 합) − (i에서 나가는 모든 항의 합)이 $DES의 유일한 문법이다.

A. Formal Definition

$DES 블록은 비선형 ADVAN(ADVAN6, 8, 9, 13) 사용 시 각 구획의 질량 변화율을 미분방정식으로 서술하는 제어구문이다. 구획 i의 변화율은 DADT(i)로 표현하며, 구획 i의 현재 질량은 A(i)로 참조한다. $DES 내부의 시간 변수는 연속형 T를 사용한다.

⚠ C-2. Plausible Fallacy — Apex Concept 전용

오류의 구체적 형태: 2구획 경구 모델에서 문헌의 $k_{12}$(central→peripheral)를 그대로 NONMEM K12로 코딩하는 오류. NONMEM에서 K12는 "구획1(Depot)→구획2(Central)" 이동을 의미하므로, 문헌 $k_{12}$는 실제로 K23으로 코딩해야 한다.

; ❌ 잘못된 코드 — K12 = 문헌 개념으로 central→peripheral을 의미하려 했으나
;    NONMEM은 K12 = 구획1(Depot)→구획2(Central)로 해석!
$DES
    DADT(1) = -KA*A(1)
    DADT(2) =  KA*A(1) - K12*A(2) - K20*A(2) + K21*A(3)  ; K12 의미 충돌!
    DADT(3) =  K12*A(2) - K21*A(3)

; ✅ 올바른 코드 — NONMEM 구획 번호에 맞게 K23/K32 사용
$DES
    DADT(1) = -KA*A(1)
    DADT(2) =  KA*A(1) - K23*A(2) - K20*A(2) + K32*A(3)  ; K23 = 구획2→구획3
    DADT(3) =  K23*A(2) - K32*A(3)                       ; K32 = 구획3→구획2

나비효과: 오류 코드에서 NONMEM은 K12를 사실상 두 번째 KA로 추정하며, 말초 구획이 모델에서 소멸한다. 피팅은 수렴하나 IIV(eta_CL)가 비정상적으로 팽창하고, CWRES vs TIME에서 초기 시점(+), 말기 시점(-)의 이중 부호 편향이 나타난다.

GOF 시그니처 이름: "Half-Life Split Sign" (이중 부호 분리 패턴) — 분포상(α phase)을 무시하고 단일 지수로 적합했을 때의 전형적 징후.

$DES 검증 체크리스트

모든 구획의 DADT(i)를 합산하면 순 유출(소실)만 남아야 한다:

$$\sum_i \text{DADT}(i) = -k_{20} \cdot A(2)$$

이 관계가 성립하지 않으면 어딘가에 부호 오류 또는 파라미터 누락이 있다.

선형 vs 비선형 구현 비교 — 동일 모델 이중 구현
개념 Big Idea: 거장의 시각

왜 치명적으로 중요한가: 동일한 2구획 경구 모델을 ADVAN5와 ADVAN6 두 가지로 구현하여 결과를 비교하는 것은 코딩 오류를 잡는 가장 강력한 검증 수단이다 — 두 구현이 다른 파라미터를 반환한다면 어딘가에 코딩 오류가 있다는 증거다.

체화해야 할 단 하나의 직관: ADVAN5는 "정답을 먼저 알고 대입하는 방식(행렬 지수)", ADVAN6는 "한 걸음씩 수치 적분하는 방식"이다. 1차 이동만 있다면 ADVAN5가 더 빠르고 안정적이다.

이 직관을 머릿속에 박고 아래를 읽어라: 비선형성이 없다면 ADVAN5/7을 써라 — 같은 모델을 더 빠르고 안정적으로 추정한다.

B. 동일 2구획 경구 모델: ADVAN5 vs ADVAN6 (PIPET 코드5.1 & 5.2)

; ════ ADVAN5 구현 — $DES 없음, K_mn 마이크로상수로 정의 ════
$SUBROUTINE ADVAN5           ; TOL 불필요
$MODEL
    COMP(DEPOT, DEFDOSE)
    COMP(CENTRAL, DEFOBS)
    COMP(PERIPH)
$PK

복잡 흡수 모델 패턴 — 병렬 흡수와 분수 파라미터
개념 Big Idea: 거장의 시각

왜 치명적으로 중요한가: 약물의 실제 흡수는 단순 1차가 아닌 경우가 압도적으로 많다 — 경구 서방형, 피하 흡수, 폐 흡수에서 병렬·순차 흡수가 나타나며, 이를 단순 ADVAN2로 적합하면 Tmax와 흡수 AUC를 체계적으로 오추정한다.

체화해야 할 단 하나의 직관: 두 depot에 전체 용량을 각각 넣되 $F_1 + F_2 = 1$로 제약하면 — 분수(fraction)가 파라미터가 된다. 이것이 "추정된 분수 흡수" 패턴의 핵심이다.

이 직관을 머릿속에 박고 아래를 읽어라: 두 depot에 동량을 넣고 생체이용률 파라미터로 분배 비율을 추정하는 것이 병렬 흡수 모델의 표준 데이터셋 설계다.

구조적 동형성 지도

통찰의 서막 — 연립 ODE 시스템의 보편성

공통 원리: $\frac{d\mathbf{A}}{dt} = \mathbf{K}\mathbf{A}$ 형태의 연립 선형 ODE는 질량·에너지·개체 수가 네트워크 간을 이동할 때 나타나는 보편적 수학 구조다. 이 구조는 약동학, 화학 반응, 역학적 감염 모델, 전기 회로, 생태계 등 거의 모든 동적 시스템에 공통으로 등장한다.

왜 이 형태인가: 임의의 구획 i에서 유출 속도가 현재 양 A(i)에 비례하는 1차 과정이라면, 각 구획의 순간 변화율은 "비례 상수 × 현재 양의 선형 조합"으로 표현될 수밖에 없다. 이 비례 관계가 K 행렬을 통해 연립되면 $\mathbf{K}\mathbf{A}$ 구조가 나타난다.

예측 전이: RC 사다리 회로의 시상수를 이해하면, 다구획 PK 모델에서 α·β 반감기의 물리적 의미와 초기 분포상이 왜 나타나는지를 먼저 예측할 수 있다.

🔌 RC 회로 → PK: 시상수로 반감기 예측
1

RC 직렬 회로에서 커패시터가 방전할 때 전압은 $V(t) = V_0 e^{-t/RC}$로 감소한다. 시상수 $\tau = RC$는 전압이 $V_0/e ≈ 36.8\%$로 감소하는 시간이다.

2

공통 수학 구조: $dV/dt = -V/RC$ ↔ $dA/dt = -(CL/V)A$. 여기서 $RC \leftrightarrow V/CL$이다.

3

따라서 1구획 PK에서 반감기 $t_{1/2} = 0.693 \cdot V/CL$은 RC 회로에서 $t_{1/2} = 0.693 \cdot RC$와 수학적으로 동일하다. 2구획 말초 구획을 추가하면 RC 사다리의 단계 추가와 동형이며, 새로운 시상수(β 반감기)가 나타난다.

🦠 SIR 모델 → PK: 비선형성의 필연
1

SIR 역학 모델에서 감염자 I의 변화율은 $dI/dt = \beta SI - \gamma I$로, S(감수성자)와 I(감염자)의 곱 항이 포함된 비선형 ODE다.

2

공통 구조: 두 구획의 양(S, I)이 곱으로 결합된 비선형 전이 → NONMEM에서 ADVAN5/7(선형 ADVAN)으로 표현 불가 → $DES 비선형 ADVAN 필수.

3

PK에서 Michaelis-Menten 소실($V_{max} \cdot C/(K_m+C)$)이나 TMDD의 수용체-약물 결합($k_{on} \cdot R \cdot C$)도 동일한 비선형 전이 구조 → ADVAN6 $DES 필수임을 SIR 예로부터 직접 예측할 수 있다.

Scenario 2 · Parent-Metabolite TDM 컨텍스트

비선형 대사체 동시 추정 — ADVAN6 $DES 임상 적용

경구 투여 후 모약(parent)과 대사체(metabolite) 농도를 동시에 TDM하는 약물에서, 포화 대사(Michaelis-Menten)를 ADVAN6 $DES로 구현한 모델 기반 용량 조절 프로토콜.

; ── Parent-Metabolite 비선형 모델 (Owen Ch.9, §9.4.6 기반) ──
$SUBROUTINE ADVAN6 TOL=4
$MODEL
    COMP(DEPOT,    DEFDOS)     ; 구획1: 흡수 구획
    COMP(CENTPRNT, DEFOBS)     ; 구획2: 모약 중심 구획
    COMP(CENTMETB)             ; 구획3: 대사체 중심 구획
$PK
    K20  = THETA(1)*EXP(ETA(1)); V2=THETA(2)*EXP(ETA(2)); KA=THETA(3)
    VMAX = THETA(4);             KM=THETA(5)
    K30  = THETA(6);             V3=THETA(7); MPR=THETA(8)
    S2=V2/1000; S3=V3/1000
$DES
    DADT(1) = -KA*A(1)
    DADT(2) =  KA*A(1) - K20*A(2) - (VMAX*A(2))/(KM+A(2))
    DADT(3) = -K30*A(3) + ((VMAX*A(2))/(KM+A(2)))*MPR
$ERROR
    IF(CMT.EQ.2) TYPE=0          ; 모약 관찰
    IF(CMT.EQ.3) TYPE=1          ; 대사체 관찰
    Y = F*EPS(1)*(1-TYPE) + F*EPS(2)*TYPE

진단 병리학 — GOF 시그니처

🔴 시그니처 1: "Half-Life Split Sign" — K 인덱스 혼동 시그니처
GOF 플롯 패턴

CWRES vs TIME: 초기 시점(t < 2t½) 체계적 양(+) 편향, 말기 시점(t > 2t½) 체계적 음(−) 편향. CWRES vs IPRED: 고농도 구간 양편향, 저농도 구간 음편향.

기계론적 원인

$DES에서 K12(문헌: central→peripheral)를 NONMEM K12(Depot→Central)로 코딩 → 말초 구획 소멸 → 2구획 분포상(α phase)이 모델에 없으므로 1구획으로 적합 → 초기 고농도를 과소추정, 말기 저농도를 과대추정.

해결 방향

$DES의 K 인덱스를 NONMEM 구획 번호 기준으로 전면 재검토. ADVAN5 이중 구현으로 교차 검증 수행. η-shrinkage 과도 증가 확인.

🔴 시그니처 2: "TOL Instability Spiral" — TOL 과소 설정 수렴 실패
GOF 플롯 패턴

NONMEM 출력: "MINIMIZATION TERMINATED" 또는 OFV가 반복마다 진동. 추정 완료 시에도 SE가 비정상적으로 크거나 $COVARIANCE step 실패.

기계론적 원인

비선형 ADVAN(ADVAN6)에서 TOL이 SIGDIGIT와 같거나 작게 설정 → 수치 적분의 내부 계산 정밀도 부족 → 목적함수(OFV) gradient 계산 불안정 → 최적화 실패.

해결 방향

TOL을 SIGDIGIT+2로 증가. stiff 모델 의심 시 ADVAN8(Gear)으로 전환. MAXEVAL=1로 ADVAN6/8 속도 비교 후 선택. ε-shrinkage로 데이터 정보량 점검.

🔴 시그니처 3: "Depot Doubling Bias" — 병렬 흡수 F₁ 제약 누락
GOF 플롯 패턴

DV vs IPRED: 모든 농도 구간에서 체계적 하방 편향(IPRED가 DV보다 2배 높음). 추정된 CL과 V가 문헌 대비 약 50% 수준.

기계론적 원인

병렬 흡수 모델에서 두 dosing record에 전체 용량을 넣되 F₁+F₂=1 제약 없이 구현 → 총 투입량이 실제 용량의 2배 → CL과 V가 2배로 추정되어야 OFV가 최소화 → 파라미터 전면 2배 편향.

해결 방향

$PK에 반드시 F2=1-F1 (또는 동등한 보완 관계) 추가. 단일 데이터 재검증 시 F₁ 초기값 0.3~0.7 범위 설정. η-shrinkage와 NPDE 분포로 추가 진단.

Q1. $SUBROUTINE ADVAN6 TOL=4에서 TOL=4가 의미하는 것은? $ESTIMATION의 SIGDIGIT=3와 어떤 관계인가?
★★▼

TOL은 각 구획 내부값(질량, 농도, 효과) 계산에 필요한 유효 자릿수(NRD, Number of Required Digits)를 의미한다. TOL=4는 수치 적분에서 4자리 유효숫자를 유지하라는 지시다.

일반 규칙: TOL = SIGDIGIT + 1 또는 2. SIGDIGIT=3이면 TOL=4 또는 TOL=5로 설정한다. TOL이 너무 낮으면 수치 부정확으로 OFV gradient가 불안정해지고, 너무 높으면 실행 시간이 과도하게 증가한다.

TOL은 최종 파라미터 추정값의 유효 자릿수가 아니라, 적분 계산 내부 정밀도임에 주의.

🔍 다음 깊이 질문: TOL을 SIGDIGIT보다 낮게 설정하면 NONMEM은 어떤 출력 메시지를 내는가?

Q4. [적용] 병렬 흡수 모델에서 두 dosing record에 각각 전체 용량(300mg)을 입력하는 이유와 F₁+F₂=1 제약이 없을 때의 오류를 설명하라.

B. Dependencies — 대충 넘겼을 때 발생하는 실패 모드

실패 모드 1: K 인덱스 혼동 → 침묵의 모델 붕괴. $DES에서 구획 번호와 K 인덱스의 관계를 체득하지 못한 모델러는 문헌의 K12(central→peripheral)를 NONMEM K12(depot→central)로 혼용한다. NONMEM은 경고 없이 수렴하고 말초 구획이 소멸한 채로 파라미터를 반환한다. 이 오류는 CWRES 이중 부호 분리 패턴으로만 사후 발견된다 — 규제 제출 후 발각되면 전체 모델 재개발이 요구된다.
실패 모드 2: 복잡 흡수 모델 데이터셋 설계 오류 → 파라미터 2배 편향. 병렬 흡수에서 F₁+F₂=1 제약의 필연성을 이해하지 못하면, 두 depot에 제약 없이 전체 용량을 입력하여 CL과 V가 실제값의 50%로 추정된다. 항암제에서 이 오류는 독성 임계치 계산 실패로 이어진다.

C. Professional Moat

NONMEM을 실행하고 파라미터를 추출하는 것은 이미 자동화 가능하다. $DES 코드 구조와 ODE 수치 적분의 수학을 진정으로 내면화한 모델러만이 다음을 할 수 있다:

수렴 실패 앞에서: NONMEM의 "MINIMIZATION TERMINATED" 메시지를 보고 TOL 문제인지, ODE 부호 오류인지, 파라미터 식별성 문제인지를 에러 메시지 한 줄에서 구분하는 진단 논리를 즉시 실행할 수 있다.

CWRES 편향 앞에서: Tmax 체계적 과소추정 패턴을 보고 "단순 1차 흡수의 구조적 misspecification"으로 진단하고 병렬 흡수 모델 전환의 필요성을 데이터 없이 판단할 수 있다 — 이것은 $DES 구조와 흡수 ODE의 형태가 실제 약물 동태에 어떻게 대응하는지를 체득한 사람만의 능력이다.

신규 생물학적 현상 앞에서: 기존 특수 ADVAN이 없는 새로운 기전(수용체 매개 내재화 + 리간드 재순환 + 표적 재생성)을 처음 마주쳤을 때, 생물학적 프로세스를 ODE로 번역하고 $DES 블록으로 즉시 구현할 수 있다. 이 능력은 특수 ADVAN 사용법 암기로는 절대 도달할 수 없으며, ODE와 $DES 구문이 1:1로 대응한다는 구조적 이해에서만 나온다.
