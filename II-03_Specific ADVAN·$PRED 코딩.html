Big Idea

§2 — 개념 해부 카드

① NONMEM 시스템 3요소와 제어구문 레코드 구조

개념 Big Idea: 거장의 시각

NONMEM을 "실행하는" 사람과 "제어하는" 사람의 차이는 바로 이 레코드 구조를 뼈대처럼 암기하고 있는가에 달려있다. 오류 메시지가 뜰 때 어느 레코드에서 문제가 생겼는지를 즉각 추적하지 못하면, 디버깅에 몇 시간을 낭비하게 된다.

제어구문은 NONMEM에게 보내는 "작업 지시서"다 — "무엇을 분석할 것인지($PROB/$DATA/$INPUT)", "어떤 모델로 할 것인지($SUBROUTINE/$PK/$ERROR 또는 $PRED)", "어디서 시작할 것인지($THETA/$OMEGA/$SIGMA)", "어떻게 실행할 것인지($EST)", "무엇을 출력할 것인지($COV/$TABLE)"의 다섯 질문에 순서대로 답한다.

제어구문의 각 레코드는 독립적으로 존재하지 않고, 위에서 아래로 흐르는 인과적 순서를 갖는다 — 이 순서를 먼저 장착하고 각 레코드의 세부 옵션을 익히면 암기가 아닌 이해로 체화된다.

A. Formal Definition

NONMEM 시스템은 세 구성요소로 이루어진다. NM-TRAN은 사용자가 작성한 제어구문을 파싱하여 문법·형식 오류를 검사하고 NONMEM이 읽을 수 있는 형태로 변환하는 번역기다. PREDPP는 집단약동학 모델링을 위해 미리 구축된 FORTRAN 서브루틴 라이브러리로, 다양한 PK 모델 구조(ADVAN)와 파라미터화(TRANS)를 제공한다. NONMEM 엔진 자체는 비선형 혼합효과 모델의 파라미터 추정을 수행하는 최적화 알고리즘의 핵심이다.

B. Code Structure

$PROB  1-compartment oral model        ; 프로젝트 식별자
$INPUT ID TIME AMT DV MDV WT AGE SEX CLCR
$DATA  test.csv IGNORE=@              ; @ = 비숫자 행 제외
$SUBR  ADVAN2 TRANS2
$PK
  TVCL = THETA(1)
  CL   = TVCL * EXP(ETA(1))
  TVV  = THETA(2)
  V    = TVV  * EXP(ETA(2))
  TVKA = THETA(3)
  KA   = TVKA
  S2   = V/1000                        ; dose(mg)/V(L) → Cp(ng/mL)
$ERROR
  IPRED = F
  W     = SQRT(THETA(4)**2 + THETA(5)**2*IPRED**2)
  IRES  = DV - IPRED
  IWRES = IRES / W
  Y     = IPRED + W*EPS(1)
$THETA  (0,3.9) (0,52.1) (0,0.987) 1 0.2
$OMEGA  0.09  0.16
$SIGMA  1 FIX
$EST    METHOD=1 INTER MAXEVAL=9999 PRINT=5
$COV    PRINT=E
$TABLE  ID TIME DV IPRED IRES CWRES ONEHEADER NOPRINT FILE=sdtab001
[출처: Owen, Ch.3, p.31 / PIPET, Ch.3, pp.29–30]

C. Structural Necessity

이 레코드 순서가 불가피한 이유: NM-TRAN은 위에서 아래로 순차 파싱한다. $DATA/$INPUT이 $PK보다 앞에 위치해야 하는 것은, $PK 내부에서 참조하는 공변량 변수(WT, CLCR 등)가 먼저 선언되어 있어야 NM-TRAN이 undefined variable 오류 없이 컴파일할 수 있기 때문이다.

D. Assumptions & Boundary Conditions

  • 데이터 파일은 순수 ASCII 텍스트 — 워드프로세서 형식 불가
  • 한 레코드 최대 160자 (NONMEM 7+). $INPUT 변수 최대 50개
  • IGNORE=#와 조건식 IGNORE=(…)은 동시 사용 가능; ACCEPT와 조건식 IGNORE는 동시 사용 불가
  • $TABLE 파일 번호(sdtab001, patab001)가 일치해야 Xpose4에서 자동 로딩

F. Five Cognitive Layers

레이어내용
L1 형식적 수학제어구문은 $y_{ij} = f(\theta,\eta_i,t_{ij}) + g(\theta,\eta_i,t_{ij})\cdot\varepsilon_{ij}$ 를 레코드로 분해한 것. $f$=$PK, $g$=$ERROR, $\theta$=$THETA, $\eta$=$OMEGA, $\varepsilon$=$SIGMA
L2 기하학적 직관THETA 하한(0)이 음수 CL 추정을 원천 봉쇄하고, MAXEVAL이 탐색 반경을 제한한다
L3 구조적 동일성Python 클래스 정의: $PK = __init__, $ERROR = 손실함수, $ESTIMATION = model.fit()
L4 생리학적 의미$INPUT의 공변량(WT, CLCR)은 IIV의 "측정 가능한 원인" — $PK에서 TVCL 식에 연결되어 미측정 ETA 크기를 줄인다
L5 실무 투영sdtab/patab/catab/cotab 관용은 Xpose4 자동 인식용 — 규칙 위반 시 R의 xpose.data() 로딩 실패

⚡ Apex Concept특수 ADVAN + TRANS 조합 — 모델 구조와 파라미터화 선택

개념 Big Idea: 거장의 시각

임상 1상 시험의 경구 투여 2구획 데이터에 ADVAN3 대신 ADVAN4를 적용하거나, TRANS4 대신 TRANS1을 선택하면 파라미터 추정값의 생물학적 해석이 완전히 달라진다. IND 제출용 모델에서 이 선택의 근거를 방어하지 못하면 FDA Deficiency Letter의 직접적 원인이 된다.

ADVAN은 "구획 구조와 투여 경로"를 선택하는 것이고, TRANS는 "그 구조를 어떤 언어(파라미터)로 표현할 것인가"를 선택하는 것이다 — 같은 생리학적 모델이 TRANS1에서는 속도상수($k_{10}$, $k_{12}$, $k_{21}$)로, TRANS4에서는 CL·V로 말해진다.

ADVAN = 구획 설계도, TRANS = 그 설계도를 읽는 언어 — 언어마다 추정값의 해석과 초기값 설정 전략이 달라진다.

B. ADVAN4 TRANS4 — 2구획 경구 모델 완전 코드

$SUBROUTINES ADVAN4 TRANS4
$PK
  TVCL = THETA(1)             ; 집단 CL (L/h)
  TVV2 = THETA(2)             ; 집단 중심구획 V (L)
  TVV3 = THETA(3)             ; 집단 말초구획 V (L)
  TVQ  = THETA(4)             ; 집단 구획간 CL (L/h)
  TVKA = THETA(5)             ; 집단 흡수속도상수 (1/h)
  CL   = TVCL * EXP(ETA(1))  ; 개인별 CL: 로그정규 IIV
  V2   = TVV2 * EXP(ETA(2))
  V3   = TVV3 * EXP(ETA(3))
  Q    = TVQ  * EXP(ETA(4))
  KA   = TVKA * EXP(ETA(5))
  ALAG1= THETA(6)*EXP(ETA(6)); 흡수지연 (선택적)
  S2   = V2/1000              ; dose(mg) → Cp(ng/mL) 단위 통일
  ; KE, K23, K32 → TRANS4가 내부 자동 계산: k10=CL/V2, k23=Q/V2, k32=Q/V3
[출처: PIPET, Ch.4, pp.49–50] [출처: Owen, Ch.3, pp.36–39]

척도 파라미터 단위 통일 공식:

$$S_n = V \times \text{usv}, \quad \text{usv} = \frac{1}{1000} \text{ (dose:mg, V:L, Cp:ng/mL일 때)}$$ [출처: Owen, Ch.3, p.41, Eq.3.6]

ADVAN6 — 비선형 일반 ADVAN (모약물-대사체 동시 분석)

$SUBROUTINES ADVAN6 TOL=4   ; TOL 없으면 실행 자체 불가!
$MODEL
  COMP(DEPOT,   DEFDOSE)
  COMP(CENTRAL, DEFOBS)
  COMP(PERIPH)
$DES
  DADT(1) = -KA*A(1)
  DADT(2) = KA*A(1) - KE*A(2) - K12*A(2) + K21*A(3)
  DADT(3) = K12*A(2) - K21*A(3)
C-2 Plausible Fallacy — TRANS 이중 선언 함정

오류 형태: ADVAN4 TRANS4 사용 중 $PK 블록에 KE = CL/V2, K23 = Q/V2를 추가로 기술. "투명하게 모든 파라미터를 선언해야 안전하다"는 믿음에서 비롯.

나비효과: TRANS4는 이미 내부에서 이 변환을 수행하므로 이중 정의 충돌이 발생한다. 예측값 F가 잘못 계산되어 GOF 플롯에서 CWRES가 체계적 음의 편향("하강 파형 패턴")을 나타낸다.

진단 시그니처: "TRANS 이중 선언 편향 파형" — DV vs IPRED에서 대각선 위쪽에 점 집중 (모델이 농도를 과소추정).

D. Assumptions & Boundary Conditions

  • 특수 ADVAN은 1차 속도상수만 지원 → 비선형 분포에는 부적합
  • ADVAN5/7은 선형 전용 → 비선형 소실에는 ADVAN6/8/9/13 사용
  • ADVAN6에서 TOL 미지정 시 실행 중단 (에러: "TOL not specified")

레이어내용
L1 형식적 수학ADVAN4 TRANS4 내부 변환: $k_{10}=CL/V_2$, $k_{23}=Q/V_2$, $k_{32}=Q/V_3$. 2구획 ODE: $\frac{dA_2}{dt}=k_aA_1-(k_{10}+k_{23})A_2+k_{32}A_3$

③ $PRED — ADVAN을 쓰지 않는 코딩

개념 Big Idea: 거장의 시각

PK/PD 모델링에서 Emax, Sigmoid-Emax, QTc-농도 관계 등 ADVAN 체계로 표현할 수 없는 모델이 빈번하다. 이때 $PRED를 모르면 NONMEM 사용 범위가 PK로만 제한되고, PK/PD 통합 모델링에서 주도권을 잃는다.

$PRED는 NONMEM에게 "PK 패키지를 내려놓고, 내가 직접 수식을 쓸게"라고 선언하는 것이다 — 그 대가로 AMT, CMT 같은 PREDPP 약속어가 사라지고, 데이터셋에서 용량을 매 행마다 공변량처럼 반복 입력해야 한다.

$PRED는 자유도가 높지만 책임이 무겁다 — 미분방정식 풀이가 없으므로 해석해를 직접 넣어야 한다.

B. Derivation / Code Structure

Sigmoidal-Emax PD 모델 — $PRED 완전 코드 (PIPET 코드 7.1):

$PROB  Sigmoidal PD Model
$DATA  ../sigmoidal_effect.csv IGNORE=@
$INPUT ID CP RESP=DV      ; CP=혈중농도, RESP=약효(DV로 재명명)
$PRED
  E0   = THETA(1)          ; 기저효과
  EMAX = THETA(2)          ; 최대효과
  C50  = THETA(3)*EXP(ETA(1)) ; EC50 (IIV 있음)
  GAM  = THETA(4)          ; Hill 계수 (IIV 추정 어려움)
  IPRD = E0 + (EMAX-E0)*CP**GAM / (CP**GAM + C50**GAM)
  Y    = IPRD + EPS(1)     ; $ERROR 없이 여기서 Y 직접 정의
[출처: PIPET, Ch.7, p.71 코드 7.1]

ADVAN2 vs $PRED 데이터셋 구조 차이 (PIPET 그림 7.2):

ADVAN2 방식$PRED 방식
AMT=150000을 투여 행에만 한 번 기재
(PREDPP가 이후 시간 내내 "기억")		AMT(용량)을 모든 관찰 행에 공변량처럼 반복 기재
(PREDPP 적분 엔진 없으므로)
[출처: PIPET, Ch.7, p.70 그림 7.2]

C. Structural Necessity

$PRED가 이 구조를 가져야 하는 이유: PREDPP를 호출하지 않으면 NONMEM의 내부 시간 적분 엔진이 동작하지 않는다. 따라서 모든 시점에서의 예측값을 해석해로 직접 계산하고, 최종적으로 Y를 정의하여 목적함수 계산기에 전달해야 한다. $ERROR 블록의 역할(잔차 정의)을 $PRED 내부에서 동시에 수행한다.

D. Assumptions & Boundary Conditions

  • 미분방정식 형태의 모델에는 사용 불가 — 해석해 필요
  • 용량은 데이터셋 모든 관찰 행에 상수로 존재해야 함
  • $PRED와 $SUBROUTINE은 같은 제어구문에 공존 불가

F. Five Cognitive Layers

레이어내용
L1 형식적 수학1구획 경구 해석해: $C(t)=\frac{D\cdot k_a}{V(k_a-k_{10})}\left(e^{-k_{10}t}-e^{-k_a t}\right)$. $PRED의 IPRED 라인이 이 수식의 직접 FORTRAN 번역
L2 기하학적 직관$PRED 모델은 파라미터 탐색 경계가 없다 — THETA 하한 미설정 시 음수 EC50도 탐색 대상이 됨
L3 구조적 동일성$PRED = GLM 커스텀 링크 함수: Y = g(θ,x) + ε 형태를 사용자가 직접 정의
L4 생리학적 의미Hill 계수(GAM)는 수용체 협동성: GAM>1 양성, GAM=1 Langmuir 등온흡착, GAM<1 음성 협동
L5 실무 투영ICH E14 후속 지침에서 QTc-노출 관계 분석을 $PRED 기반 Hill 모델로 구현하는 것이 규제 표준화 추세

④ $ERROR 잔차변이 구조와 $THETA/$OMEGA/$SIGMA 초기값 전략

개념 Big Idea: 거장의 시각

잔차변이 구조를 잘못 선택하면 — 고농도에서 비례오차가 지배적임에도 가법오차를 적용하면 — CWRES가 농도에 비례하여 증가하는 깔때기 패턴이 나타나고, $OMEGA 추정값이 과대추정되어 모든 하위 분석(시뮬레이션, VPC)이 오염된다.

$ERROR는 "측정 오차의 성격"을 모델링하는 것이다 — 절대 오차가 농도와 무관하면 가법(additive), 비례하면 비례(CCV), 둘 다라면 혼합(combined) 모델.

저농도(검출 한계 근처)에서는 가법 오차, 고농도에서는 비례 오차가 지배적인 경우가 많다 — 이것이 혼합 잔차 모델을 권장하는 생물학적 이유다.

B. Code Structure — 주요 잔차 구조

; 1. 가법 잔차 — 저농도/절대 측정오차 일정
Y = F + EPS(1)                            ; Var(Y) = σ²

; 2. 비례(CCV) 잔차 — 고농도/상대오차 일정
Y = F*(1 + EPS(1))                        ; Var(Y) = F²·σ²

; 3. W함수 혼합 잔차 (권장 패턴 — SIGMA 1 FIX)
W     = SQRT(THETA(3)**2 + THETA(4)**2*IPRED**2)
IRES  = DV - IPRED
IWRES = IRES/W
Y     = IPRED + W*EPS(1)
; THETA(3) = 가법 SD (단위: 농도), THETA(4) = 비례 CV (무단위)
; $SIGMA 1 FIX — W가 이미 잔차 크기 조절, SIGMA=1은 정규화
[출처: PIPET, Ch.3, pp.36–37] [출처: Owen, Ch.3, pp.45–47]

$OMEGA 초기값 결정 공식:

$$\omega^2 = \left(\frac{\%CV}{100}\right)^2 \quad \Rightarrow \quad \omega^2_{CL} = (0.30)^2 = 0.09$$ [출처: Owen, Ch.3, p.53]

C. Structural Necessity

EXP(ETA) 형태의 IIV 모델이 선호되는 이유: ETA가 아무리 큰 음수여도 $CL_i = TVCL \cdot e^{\eta_i} > 0$이 항상 보장된다. 반면 비례 모델 $CL_i = TVCL \cdot (1+\eta_i)$에서 $\eta_i < -1$이면 음수 CL이 발생 — 약동학 파라미터는 물리적으로 양수만 허용되므로 지수 모델이 구조적 필수다.

D. Assumptions & Boundary Conditions

  • $OMEGA/$SIGMA 초기값은 분산(variance) 입력 — 표준편차(SD) 아님
  • $OMEGA/$SIGMA에 0 사용 시 반드시 FIX 필요 (추정 0은 경계값 오류)
  • $OMEGA BLOCK에서 FIXED 하나라도 있으면 블록 전체 FIXED 처리

§3 — 구조적 동형성 지도

공통 원리: NM-TRAN 제어구문의 레이어드 아키텍처($DATA→$PK→$ERROR→$THETA)는 소프트웨어 공학의 관심사 분리(Separation of Concerns) 원칙의 완벽한 사례이며, 동시에 통계학의 계층적 모델(hierarchical model) 구조와 1:1로 대응한다.

왜 이 형태인가: 고정효과($\theta$)·개인간변이($\eta$)·잔차($\varepsilon$)의 3층 구조는 서로 다른 변산성 원천을 분리하여 독립적으로 추정하기 위한 수학적 필연이다. 이를 단일 레코드로 뭉치면 파라미터 식별성(identifiability)이 소멸한다.

🔄 전이 질문 1: 통계학자의 Stan 지식으로 NONMEM $OMEGA BLOCK을 즉각 해석할 수 있는가? ▶
① 출발 도메인 (베이지안 통계학): Stan의 hierarchical model에서 개인별 파라미터 $\theta_i \sim \mathcal{N}(\mu, \Sigma)$로 정의할 때, $\Sigma$는 공분산 행렬로 개인간 상관을 표현한다.
② 공통 수학 구조: NONMEM의 $\eta_i \sim \mathcal{N}(0, \Omega)$에서 $\Omega$ 행렬이 정확히 Stan의 $\Sigma$에 해당한다. $\Omega$의 대각 원소 = 각 파라미터의 IIV 분산, 비대각 원소 = 두 파라미터 간 공분산.
③ 대응 현상 도출: Stan에서 CL과 V 사이의 양의 상관($\rho > 0$)을 표현하는 사람은 NONMEM에서 $OMEGA BLOCK(2) 0.09 / 0.03 0.16이 CL-V 공분산 0.03을 추정한다는 것을 즉각 이해한다.
🔄 전이 질문 2: 소프트웨어 엔지니어의 "관심사 분리" 지식으로 $PRED와 ADVAN의 Trade-off를 예측할 수 있는가? ▶
① 출발 도메인 (소프트웨어 공학): 프레임워크(Rails, Django)는 관용 패턴(ADVAN)을 제공하여 빠른 개발을 가능케 하지만, 특수 요구사항에는 커스텀 코드($PRED)가 필요하다. 프레임워크를 사용하면 "설정보다 관습(convention over configuration)"이 적용된다.
② 공통 수학 구조: 특수 ADVAN은 "프레임워크 모드" — 사전 정의된 ODE를 빠르게 호출. $PRED는 "커스텀 모드" — 사용자가 모든 로직을 직접 구현. 두 경우 모두 최종 출력(Y, 목적함수)은 동일한 NONMEM 추정 엔진에 의해 처리된다.
③ 대응 현상 도출: 소프트웨어 엔지니어는 "$PRED로 다회 투여를 구현하면 superposition을 수동으로 코딩해야 한다"는 것을 — "프레임워크 기능을 커스텀 모드에서 직접 구현해야 한다"는 원칙으로부터 즉각 예측한다.

  1. PRINT=5 출력에서 gradient 확인 — gradient가 0.000인 파라미터가 있으면 해당 파라미터가 모델에 기여하지 않음 (데이터 부족 또는 불필요한 파라미터)
  2. 최종 추정값과 초기값 비교 — 100배 이상 차이나면 "NEAR BOUNDARY" 경고 발생. 해결: NCA 또는 EDA로 초기값 재설정
  3. $COV UNCONDITIONAL 추가 — 추정 단계 성공 여부 무관하게 $COV step 강제 실행. RSE가 너무 높으면(>50%) 파라미터 삭제 또는 모델 단순화
  4. $EST NOTBT NOOBT NOSBT 추가 — 경계 검사 비활성화. 임시방편이므로 근본 원인 해결 병행 필수
  5. $OMEGA 행렬 단순화 — BLOCK → 대각, IIV 항목 순차 제거하며 수렴 확인

Diagnostic Pathology — 진단 병리학

① TRANS 이중 선언 편향 파형 (Double-declaration Bias Pattern)

GOF 플롯 패턴: DV vs IPRED에서 대각선 위쪽에 점 집중 (IPRED가 DV보다 일관되게 낮음). CWRES vs TIME에서 체계적 음의 편향 (-2 이하 다수). DV vs PRED는 정상적으로 보일 수 있음.

기계론적 원인: ADVAN4 TRANS4 사용 중 $PK에 KE=CL/V2 등을 중복 기술 → NONMEM 내부 파라미터 값이 이중으로 할당되어 예측 농도가 실제보다 낮게 계산됨.

해결 방향: $PK에서 KE, K12, K21 등 TRANS가 자동 계산하는 속도상수 라인을 모두 제거. $SUBROUTINE 가이드 참조하여 해당 TRANS가 어떤 파라미터를 자동 처리하는지 확인.

η-shrinkage: 이 오류 발생 시 η-shrinkage가 비정상적으로 높게 나타나는 경우가 많음 (>40%) — 모델이 잘못된 예측을 보상하기 위해 ETA가 극단값으로 수축.

② S2 단위 불일치 전역 편향 (Scale Mismatch Global Bias)

GOF 플롯 패턴: DV vs IPRED에서 모든 점이 대각선에서 1000배 이상 이격. OFV가 매우 크게 나타남. 추정 파라미터가 초기값에서 수렴하지 않거나 극단값으로 발산.

기계론적 원인: 투여 용량 단위(mg)와 관찰 농도 단위(ng/mL) 불일치. 예를 들어 S2=V2 대신 S2=V2/1000이 필요한데 빠뜨린 경우. NONMEM의 예측 농도가 실제 DV보다 1000배 낮게 계산됨.

해결 방향: $PK에서 S2(또는 Sn) 값을 검토. 투여 단위(AMT)와 관찰 단위(DV)를 확인하고 Sn = V × usv 공식으로 올바른 척도 파라미터를 계산. Owen Ch.3의 unit balancing 절차 참조.

ε-shrinkage: S2 오류 시 ε-shrinkage 역시 극단값 — 잔차가 모두 동일한 방향으로 치우쳐 EPS의 개인 기여가 소멸.

★★ Q3. 투여 용량 단위=mg, V2=L, 관찰 농도=ng/mL일 때 S2를 어떻게 정의해야 하는가? 수식 근거 포함. ▶
S2 = V2/1000

근거: $C = \text{Amount}/V$에서 Amount(mg), V(L) → C = mg/L. DV는 ng/mL = μg/L이므로 1 mg/L = 1000 ng/mL. 분모를 1000배 키워 단위 일치: $S2 = V2 \times (1/1000) = V2/1000$
다음 깊이 질문: dose가 μg 단위로 AMT에 기재되고 농도가 ng/mL이라면 S2는 어떻게 달라지는가?
★ Q4. 다음 $ERROR 코드에서 THETA(4)의 통계적 의미와 단위를 설명하라:
W = SQRT(THETA(3)**2 + THETA(4)**2 * IPRED**2); $SIGMA 1 FIX ▶
THETA(3): 가법 잔차의 표준편차(SD), 단위=농도 단위와 동일 (예: mg/L)
THETA(4): 비례 잔차의 변동계수(CV), 무단위(소수). THETA(4)=0.2이면 비례 잔차 CV=20%
$SIGMA를 1로 고정하여 EPS(1)의 분산=1로 표준화했으므로, W 전체가 잔차의 표준편차를 나타냄.

임상 해석: 집단 전체에서 해당 PK 파라미터의 개인별 값이 집단 대표값을 중심으로 약 ±30%의 변동을 갖는다. 동일 용량 투여 시 환자 간 AUC 변동이 약 30% 수준임을 시사.

★ Q7. ADVAN6 선택 시 $SUBROUTINE에 TOL=n이 필수인 이유를 ODE 수치 적분 관점에서 설명하라. ▶
ADVAN6는 비선형 ODE를 수치적으로 적분(Runge-Kutta 등)하여 A(n)을 계산한다. TOL은 각 적분 스텝의 허용 오차를 $10^{-\text{TOL}}$로 설정한다. 지정하지 않으면 NONMEM이 기본값을 찾지 못해 실행이 중단된다. TOL이 너무 크면 적분 정확도 저하 → 농도 곡선이 계단형 → OFV에 수치 노이즈 → 수렴 불안정. TOL=4~6이 권장된다.

Option B: ADVAN2 TRANS2 (1구획) — GOF 약간 나쁘지만 $COV step 안정

▶
수석 모델러의 판단: Option B를 선택하되, Option A 결과를 투명하게 보고한다.

논리: Phase 1 IND 제출의 1차 목적은 안전성 확인과 용량 범위 파악이다. Q와 V3의 RSE > 100%는 파라미터 불확실성 정량화 불가를 의미한다. 규제 관점에서 "$COV step 실패 = SE 없음 = 불확실성 미보고"는 용량 결정 근거 취약으로 해석된다.

A 1-compartment model was therefore selected as the base model. The adequacy of this choice will be re-evaluated with dense PK data from Phase 2."

핵심 단서: Option A 분석 결과를 sensitivity analysis로 보고서에 포함 필수. ΔOFV와 GOF를 나란히 제시하면 심사관이 판단 과정을 추적할 수 있어 오히려 신뢰도를 높인다.

B. Dependencies — 실패 모드

실패 1 — S2 오류로 공변량 분석 왜곡: S2=V2/1000을 S2=V2로 잘못 기술하면 모든 예측 농도가 1000배 과소 계산된다. 이 상태에서 공변량 분석(II-08)을 진행하면 TVCL이 실제보다 1000배 큰 값으로 추정되고, WT-CL 관계의 기울기가 완전히 왜곡된다. Phase 2/3 용량 결정이 이 값에 근거한다면 치명적 용량 과다로 이어질 수 있다.

실패 2 — TRANS 선택 오류로 파라미터 해석 불능: ADVAN4에서 TRANS1을 선택하고 보고서에 "CL = K × V2"라고 기술하면, TRANS1의 K는 $k_{10} = CL_{total}/V_2$이 아니라 단순 $k_{10}$만을 의미하므로, CL_renal과 CL_hepatic 분리가 불가능해진다. NDA의 파라미터 해석 섹션 전체가 오염된다.

C. Professional Moat

NONMEM을 실행하고 파라미터를 추출하는 것은 이미 자동화 가능하다. 이 세션의 제어구문 구조와 ADVAN/TRANS/$PRED의 코딩 원리를 진정으로 내면화한 모델러만이 다음을 할 수 있다:

  • 수렴은 됐는데 예측 농도가 DV보다 1000배 작은 상황 앞에서 — "S2 단위 불일치"를 30초 안에 코드 한 줄로 진단·수정한다. 알고리즘은 DV-IPRED 이격을 감지하지만 원인을 코드 구조와 데이터 단위의 맥락에서 연결하지 못한다.
  • $COV step이 "PARAMETER NEAR BOUNDARY"와 함께 실패한 상황 앞에서 — 파라미터 식별성 문제인지, 초기값 불량인지, 데이터 희소성인지를 gradient 출력을 읽으며 구별하고, 세 가지 해결 경로 중 가장 올바른 것을 선택한다.
  • IND 마감 전날 $PRED로 작성한 QTc-노출 모델에서 음수 Y가 발생했을 때 — FLAG=0/1 조건 코딩과 CALLFL=0 옵션의 차이를 알고 어떤 상황에서 어느 것을 쓸지 즉각 판단한다.
